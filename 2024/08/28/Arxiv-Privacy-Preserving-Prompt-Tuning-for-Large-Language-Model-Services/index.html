标题: Privacy-Preserving Prompt Tuning for Large Language Model Services
作者: Yansong Li, Zhixing Tan, Yang Liu
所属机构:

  • Yansong Li: University of Ottawa, Ottawa, Canada
  • Zhixing Tan: Zhongguancun Laboratory, Beijing, China
  • Yang Liu: Department of Computer Science and Technology, Tsinghua University, Beijing, China; Institute for AI Industry Research, Tsinghua University, Beijing, China; Institute for Artificial Intelligence, Tsinghua University, Beijing, China

Abstract

提示微调在新兴的大规模语言模型（LLM）服务场景中为用户提供了一种利用其私人数据定制LLM的高效方式。然而，私人数据的敏感性质使得在LLM服务定制中需要保护隐私。基于提示微调，我们提出了一种名为隐私保护提示微调（RAPT）的框架，为LLM服务提供隐私保证。RAPT采用本地隐私设置，允许用户使用本地差分隐私在本地对其数据进行隐私化处理。由于提示微调在直接使用隐私化数据进行训练时表现不佳，我们引入了一种新颖的隐私化标记重建任务，与下游任务联合训练，使LLM能够学习到更好的任务相关表示。尽管我们的框架相对简单，但实验表明，RAPT在各种任务中都能提供有竞争力的性能，同时为对抗者提供隐私保证。

Introduction

近年来，大规模语言模型（LLMs）（Devlin等，2018；Brown等，2020）取得了巨大的成功。随着LLM规模的不断增长，提示微调方法（Li和Liang，2021；Lester等，2021）作为一种简单且参数高效的解决方案，逐渐成为将LLM引导到下游任务的有效工具。这种方法不仅能实现有竞争力的性能，还支持多任务推理（Li和Liang，2021）。然而，LLM的庞大规模也使得其对个人用户来说部署和运行成本很高。因此，最近的LLM通常作为云服务发布，并经常提供提示微调API，供用户定制LLM，例如最近发布的NVIDIA NeMo LLM服务。

虽然提示微调API为用户提供了定制和使用LLM服务的高效方式，但由于私人数据的敏感性，在使用LLM服务时会引发对私人数据泄露的担忧。为了使用提示微调定制和利用LLM服务，用户需要将他们的数据上传到服务提供商。然而，众所周知，输入文本甚至嵌入表示都可能向各种对手泄露私人信息（Coavoux等，2018）。因此，在使用和定制LLM时提供隐私保护，以使LLM服务可信赖，变得非常重要。

然而，在LLM服务的背景下进行隐私保护具有挑战性。首先，在LLM服务的背景下为私人数据提供隐私保障并不简单。现有关于LLM隐私保护的研究（Yu等，2021a；Shi等，2021；Anil等，2021；Hoory等，2021；Li等，2021；Shi等，2022）主要集中于中心化的隐私设置，依赖服务提供者保护用户免受隐私泄露。在诚实但好奇的服务提供者或中间窃听者的情况下，这种设置可能不令人满意（Lyu等，2020）。其次，实施隐私保护不可避免地会降低下游任务的性能，这被称为隐私-效用权衡。Yang和Liu（2022）发现提示微调缺乏鲁棒性，因此在提供隐私保护时，使用提示微调来定制LLM可能会遭遇不可接受的性能下降。

为了解决上述挑战，我们提出了隐私保护提示微调（RAPT）框架，用于在保护隐私的同时定制和利用LLM服务。为实现隐私保护，RAPT采用本地隐私设置（Lyu等，2020；Qu等，2021），用户在发布数据之前在本地对数据应用隐私机制。具体而言，RAPT使用文本到文本的隐私化（Feyisetan等，2020），基于局部差分隐私（LDP）（Evfimievski等，2003）。为了减轻隐私保护带来的性能下降，我们提出了一种新颖的隐私化标记重建任务，基于最近的发现，即掩码语言建模目标可以学习可分离的深度表示（Voita等，2019；Mamou等，2020）。隐私化标记重建的目标是从LLM表示中恢复隐私化特殊标记序列的原始内容，这一灵感来自掩码语言建模的成功（Devlin等，2018）。与标准提示微调不同，RAPT在下游任务和隐私化标记重建任务上共同训练提示。通过这种方式，RAPT结合了提示微调的轻量和模块化的优点，同时通过隐私化标记重建提供了更高的效用（即性能）。图1展示了我们的RAPT框架的概述。

我们进行了模拟隐私攻击，并在三项不同的大规模语言模型任务上进行实验，以研究我们提出的框架的有效性。实验表明，RAPT在提供隐私保护的同时，在不同任务和LLM上也能实现具有竞争力的性能。

Related Work

最近关于大规模语言模型（LLM）隐私保护的研究大致可以分为集中式方法和本地方法。

集中式方法：大多数现有的研究聚焦于集中式隐私设置，这依赖一个中央数据管理员来保护数据免受隐私泄露。有许多研究探讨如何训练具有隐私保护功能的LLM（Carlini等人，2021；Hoory等人，2021；Li等人，2021；Yu等人，2021b），这些内容超出了本工作的范围。Kerrigan等人（2020）使用差分隐私微调来保护用于微调公共语言模型的私有数据。Yu等人（2021a）研究了在微调阶段使用轻量级微调方法（如适配器和PREFIX-TUNING）来保护隐私数据。我们的设置不同于所有这些工作。我们研究如何在使用LLM服务时保护用户的私人数据，并且不假设中央数据管理员的存在。

本地方法：本地方法提供了更强的隐私保护，但代价是效用的损失（即下游任务的性能下降）。Lyu等人（2020）研究了如何在本地隐私保护的情况下，通过差分隐私神经表示方法来维护模型的效用。他们只考虑了推理阶段的隐私保护，而没有涉及微调阶段。Qu等人（2021）提出了一种隐私约束微调（PCF）方法，用于在微调和推理阶段提供隐私保护。他们的方法需要在隐私化数据上微调整个模型，这对于大型语言模型来说是很昂贵的。我们的方法同样采用了本地隐私设置，支持在微调和推理阶段的隐私保护，但更为轻量化。我们还引入了一种新颖的隐私化标记重建任务

Approach

在本节中，我们详细描述了RAPT。我们首先介绍文本到文本的隐私化方法，这是用于用户在本地对数据进行隐私化处理的。然后，我们描述了如何使用我们的提示方法，通过隐私化数据定制和利用LLM服务。

3.1 文本到文本隐私化

使用LLM服务时可能会面临诸如窃听攻击等问题。因此，采用本地隐私设置是有利的，这样用户可以在他们的设备上本地进行数据隐私化处理。

由于大多数LLM使用文本到文本的接口，我们在我们的框架中采用了文本到文本的隐私化方法（参考Feyisetan等人，2020）。文本到文本的隐私化基于dX-隐私（Chatzikokolakis等人，2013），这是一种基于距离的局部差分隐私的松弛形式，广泛用于保护文本内容的隐私（Feyisetan等人，2020；Qu等人，2021）。形式上，对于给定的输入集X和输出集Y，dX是定义在X上的一个距离函数。一个随机机制M: X → Y 被称为满足dX-隐私，当且仅当对于任何x ∈ X 和 x′ ∈ X，M(x)和M(x′)的输出分布满足以下约束：
[
\frac{P(M(x) = y)}{P(M(x′) = y)} \leq e^{\eta dX(x, x′)}, \forall y \in Y,
]
其中η ≥ 0是隐私参数，用于控制隐私保护的程度。

为了应用文本到文本的隐私化，我们首先需要一个嵌入模型将词映射到向量。给定一个序列x = [x1, …, xn]和一个嵌入模型E ∈ R^{|V|×d}，其中|V|和d分别是词汇表的大小和嵌入的维度，嵌入模型将x映射为一个向量序列[x1, …, xn]。假设使用L2距离作为距离函数，对一个词嵌入xt ∈ R^d应用具有隐私参数η的dX，可以通过向xt添加随机噪声z ∼ exp(−η∥z∥)来实现（Wu等人，2017）。令z = lv，采样z等同于先采样标量l ∼ Γ(d, \frac{1}{η})，然后从单位球空间Bd中均匀采样向量v。因此，xt的隐私化表示M(xt)可以形式化描述为：
[
M(xt) = xt + z.
]
接下来，我们用嵌入空间中最接近xt的词x′t替换xt：
[
x′t = \arg\min_k ∥wk − M(wt)∥.
]
通过重复替换序列x中的词，我们得到x的隐私化文本版本M(x)。

3.2 LLM定制化

由于提示微调（Prompt Tuning）轻量且模块化，因此它是定制LLM服务的合适选择。我们假设用户通过上传经过隐私化的训练数据到服务提供商，使用提示微调API来定制LLM服务。

3.2.1 提示微调

提示微调使用连续向量来调整LLM，采用梯度下降来学习（Li和Liang, 2021; Lester等人, 2021）。提示微调有很多变种，例如PREFIX-TUNING（Li和Liang, 2021）、PROMPT TUNING（Lester等人, 2021）和P-TUNING（Liu等人, 2021）。为了方便描述，我们采用Lester等人（2021）的符号表示法，但其他变种同样适用于我们的方法。

设f表示LLM服务的主干模型。提示P ∈ R是一个连续向量序列[p1,…,pN]，其中pi ∈ Rh是一个可调节的虚拟标记嵌入，h是主干LLM的隐藏层大小。给定一个输入序列x = [xi,…,xn]，提示被添加到X ∈ Rn×h之前，其中X = [x1,…,xn]是词嵌入序列，x ∈ Rh。然后，我们使用LLM获得一个激活序列：

[
H = f(JP;XK), (4)
]

其中JP; X K表示序列P和X的拼接，H ∈ Rn×h是激活序列。H通常通过语言建模头用于预测任务相关的标签（Liu等人, 2021）。

3.2.2 隐私化标记重建

不幸的是，直接在隐私化数据上进行提示微调会显著降低LLM在下游任务上的性能，即使隐私保护较弱，这表明提示微调对文本到文本隐私化引入的随机扰动非常敏感。

最近关于语言模型表示的研究（Voita等人, 2019; Mamou等人, 2020）发现，掩码语言建模目标（Devlin等人, 2018）有助于学习可分离的深度表示。受到这些发现的启发，我们提出了一种新颖的隐私化标记重建任务，该任务与下游任务联合训练。我们期望通过隐私化标记重建的训练可以帮助LLM学习更好的任务相关表示，从而在隐私保护的前提下提高LLM在下游任务上的性能。

隐私化标记重建类似于掩码语言建模。然而，在隐私化输入中重建一个词是不可能的，因为该词可能包含敏感信息。为了解决这个问题，我们在训练数据中的每个输入前添加一系列标记，我们称之为“普通标记”。普通标记由任意选择的标记组成，因此可以安全地发送给服务提供商，而无需担心隐私泄露。图2提供了RAPT在LLM定制化过程中的概述。

形式上，令k = [k1,…,km]表示普通标记。给定一组训练样本D = {⟨xi, yi⟩|i = 1,…,|D|}，其中|D|表示训练数据的大小。用户首先将k添加到每个输入xi前，然后通过文本到文本隐私化获得Jk; xiK的隐私化版本M(Jk; xiK) = [k1′ ,…,km′ ,x′1,…,x′n]。设z = M(Jk; xiK)，LLM用输入z和提示P生成一个激活序列G：

[
G = f(JP;ZK), (5)
]

其中Z是z的词嵌入序列，G ∈ R(m+n)×h。

得到G后，我们使用G中的前m个向量G1:m = [g1,…,gm]来重建普通标记。为此，我们引入了一个由两层线性层组成的额外重建头部。预测第i个普通标记ki的概率分布为：

[
pi = \text{softmax}(W1W2gi), (6)
]

其中W1∈R|T|×c和W2∈Rc×h是重建头部的参数，c是重建头部的隐藏大小，|T|表示重建头部的词汇表大小。隐私化标记重建任务的损失函数为：

[
L_{\text{rec}} = -\sum_{i=1}^{m} \log pi[ji], (7)
]

其中ji表示重建头部词汇表中ki的索引，pi[ji]表示向量pi中的第ji个标量。请注意，重建头部的词汇表不需要与LLM的词汇表相同。

剩下的n个激活Gn:m+n用于预测任务相关的标签。假设下游任务使用交叉熵损失，yi ∈ {0, 1}|C|是与yj关联的标签，表示第j个类别的标签，|C|是类别的数量。下游任务的目标函数形式上描述为：

[
p = g(Gm:m+n), (8) \quad
L_{\text{task}} = -\sum_{i=1}^{|C|} yi \log p[i], (9)
]

其中g是语言建模头部的函数。因此，训练RAPT的损失函数为：

[
L = L_{\text{task}} + L_{\text{rec}}. (10)
]

在推理阶段，用户端同样应用文本到文本隐私化以保护私人信息，并从LLM服务获得预测。我们注意到，在推理阶段不使用重建头部，因此训练后可以丢弃{W1, W2}。
