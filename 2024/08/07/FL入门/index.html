Communication-Efficient Learning of Deep Networksfrom Decentralized Data

标题: Communication-Efficient Learning of Deep Networks from Decentralized Data

作者:

  • H. Brendan McMahan
  • Eider Moore
  • Daniel Ramage
  • Seth Hampson
  • Blaise Agüera y Arcas

机构: Google, Inc., 651 N 34th St., Seattle, WA 98103 USA

发表会议: AISTATS 2017

关键词: 联邦学习, 分布式深度学习, 隐私保护, 数据中心, non-IID, 模型平均

Abstract

现代移动设备可以访问大量适合训练模型的数据，这些数据反过来可以极大地改善设备上的用户体验。例如，语言模型可以改进语音识别和文本输入，而图像模型可以自动选择优质照片。然而，这些丰富的数据往往具有隐私敏感性、数量庞大或两者兼有，这可能会阻止将数据记录到数据中心并使用传统方法进行训练。我们倡导一种替代方案，将训练数据分散存储在移动设备上，通过汇总本地计算的更新来学习共享模型。我们称这种去中心化的方法为Federated Learning。

我们提出了一种基于迭代模型平均的联邦学习深度网络的实用方法，并进行了广泛的实证评估，考虑了五种不同的模型架构和四个数据集。这些实验表明，该方法对不平衡和non-IID数据分布具有鲁棒性，这些特性是此设置的定义特征。通信成本是主要限制因素，我们显示与同步SGD相比，所需的通信轮数减少了10-100倍。

Introduction

越来越多的人将手机和平板电脑作为主要的计算设备[30, 2]。这些设备上的强大传感器（包括摄像头、麦克风和GPS），结合它们经常被携带的事实，使得它们可以访问前所未有的大量数据，其中很多都是私密的。基于这些数据训练的模型有望通过支持更智能的应用来极大地提高可用性，但数据的敏感性意味着将其存储在集中式位置存在风险和责任。

我们研究了一种学习技术，使用户能够共同享受从这些丰富数据中训练的共享模型的好处，而无需集中存储数据。我们将这种方法称为Federated Learning，因为学习任务是通过一个松散联合的参与设备（我们称之为客户端）来完成的，这些设备由一个中央服务器协调。每个客户端都有一个本地训练数据集，这些数据集从未上传到服务器。相反，每个客户端计算对服务器维护的当前全局模型的更新，并且只传输这个更新。这是2012年白宫关于消费者数据隐私的报告[39]中提出的集中收集或数据最小化原则的直接应用。由于这些更新是为了改进当前模型，因此一旦应用这些更新，就没有理由存储它们。

这种方法的主要优势是将模型训练与直接访问原始训练数据的需求分离开来。显然，仍需要对协调训练的服务器有一定的信任。然而，对于训练目标可以基于每个客户端可用的数据来指定的应用，联邦学习可以通过将攻击面限制到设备而不是设备和云，大大降低隐私和安全风险。

我们的主要贡献是：1）确定了在移动设备的分散数据上进行训练的问题是一个重要的研究方向；2）选择了一种可以应用于这种环境的简洁实用的算法；3）对所提方法进行了广泛的实证评估。更具体地说，我们介绍了FederatedAveraging，该算法结合了每个客户端的本地SGD和服务器的模型平均。我们对该算法进行了广泛的实验，证明它对不平衡和non-IID数据分布具有鲁棒性，并且可以减少训练分散数据的深度网络所需的通信轮数几个数量级。

Federated learning
联邦学习理想的问题具有以下属性：1) 在移动设备上的真实世界数据进行训练，相较于在数据中心使用代理数据进行训练具有明显优势。2) 这些数据具有隐私敏感性或数据量大（相对于模型的大小），因此不应仅仅为了模型训练而将其记录到数据中心（符合集中收集原则）。3) 对于监督任务，可以从用户交互中自然推断出数据的标签。(例如用户输入的文本可以用于训练语言模型，用户在照片应用中的操作（如删除、分享、查看照片）可以为图像分类模型提供标签。)

许多驱动移动设备上智能行为的模型都符合上述标准。以两个例子为例，我们考虑图像分类，例如预测哪些照片在未来可能会被多次查看或共享；以及语言模型，用于改进触屏键盘上的语音识别和文本输入，提升解码、下一个词预测，甚至是整句回复的能力[10]。这些任务的潜在训练数据（用户拍摄的所有照片和在移动键盘上输入的所有内容，包括密码、URL、消息等）可能具有隐私敏感性。这些数据的分布也可能与容易获取的代理数据集有很大不同：聊天和短信中的语言使用通常与标准语言语料库（例如维基百科和其他网络文档）大不相同；用户在手机上拍摄的照片也可能与典型的Flickr照片大不相同。最后，这些问题的标签是直接可用的：输入的文本可以用于语言模型的自标签，照片的标签可以通过用户在照片应用中的自然交互（哪些照片被删除、共享或查看）来定义。

这两类任务都非常适合于神经网络学习。对于图像分类，前馈深度网络，特别是卷积网络，已知能提供最先进的结果[26, 25]。对于语言建模任务，循环神经网络，特别是长短期记忆网络（LSTM），已经取得了最先进的结果[20, 5, 22]。

Privacy
与在数据中心训练持久化数据相比，联邦学习在隐私保护方面具有明显优势。即使是“匿名化”的数据集，通过与其他数据联合，仍然可能使用户隐私面临风险[37]。相比之下，联邦学习中传输的信息是改进特定模型所需的最小更新（当然，隐私优势的强度取决于更新的内容）1。更新本身是（且应该是）临时的。它们包含的信息永远不会超过原始训练数据（根据数据处理不等式），并且通常包含的要少得多。此外，聚合算法不需要更新的来源，因此更新可以通过混合网络（如Tor）或可信的第三方在没有识别元数据的情况下传输。我们在文章末尾简要讨论了将联邦学习与安全多方计算和差分隐私相结合的可能性。

Federated Optimization
我们将联邦学习中的优化问题称为联邦优化，以此与分布式优化进行联系和对比。联邦优化具有几个关键特性，使其不同于典型的分布式优化问题：

  • Non-IID：给定客户端的训练数据通常基于特定用户的移动设备使用情况，因此任何特定用户的本地数据集不会代表总体分布。
  • Unbalanced：同样，一些用户会比其他用户更频繁地使用服务或应用程序，导致本地训练数据量的差异。
  • Massively distributed：我们预计参与优化的客户端数量远大于每个客户端的平均样本数。
  • Limited communication：移动设备经常离线或处于慢速或昂贵的连接状态。

在这项工作中，我们强调了优化中的非IID和不平衡特性，以及通信约束的关键性。一个部署的联邦优化系统还必须解决大量实际问题：客户端数据集随着数据的增加和删除而变化；客户端可用性与本地数据分布复杂相关（例如，美国英语用户的手机可能在与英国英语用户不同的时间插入电源,这影响了设备的可用性）；以及从不响应或发送损坏更新的客户端。

这些问题超出了当前工作的范围；相反，我们使用一个适合实验的受控环境，但仍然解决了客户端可用性以及不平衡和非IID数据的关键问题。我们假设一个同步更新方案，按通信轮次进行。存在一组固定的K个客户端，每个客户端都有一个固定的本地数据集。在每轮开始时，随机选择C个客户端，服务器将当前全局算法状态发送给这些客户端（例如当前模型参数）。我们只选择一部分客户端以提高效率，因为我们的实验表明，超过某个点后增加更多客户端的回报递减。每个选定的客户端然后根据全局状态和其本地数据集执行本地计算，并将更新发送给服务器。服务器然后将这些更新应用到其全局状态，过程重复进行。

虽然我们专注于非凸神经网络目标，但我们考虑的算法适用于任何形式的有限和目标

[
\min_{w\in\mathbb{R}^d}f(w)\quad\mathrm{~where~}\quad f(w)\overset{\mathrm{def}}{\operatorname*{=}}\frac1n\sum_{i=1}^nf_i(w).
]

对于一个机器学习问题，我们通常取(f_i(w) = l(x_i, y_i; w))，即使用模型参数(w)对样本( (x_i, y_i) )进行预测的损失。我们假设数据分配在K个客户端上，( P_k )是客户端k的数据点索引集合，( n_k = |P_k| )。因此，我们可以将目标重写为

[
f(w)=\sum_{k=1}^K\frac{n_k}{n}F_k(w)\quad\text{where}\quad F_k(w)=\frac1{n_k}\sum_{i\in\mathcal{P}_k}f_i(w).
]

如果分区(P_k)是通过将训练样本均匀随机分配到客户端形成的，那么我们会有(E_{P_k} [F_k (w)] = f (w))，其中期望值是分配给固定客户端k的样本集合的期望值。这是分布式优化算法通常假设的独立同分布（IID）假设；我们称这种情况下的Fk是对f的任意糟糕近似为非IID设置。

在数据中心优化中，通信成本相对较低，而计算成本占主导地位，最近的重点是使用GPU降低这些成本。相比之下，在联邦优化中，通信成本占主导地位——我们通常会受限于1 MB/s或更低的上传带宽。此外，客户端通常只有在充电、插入电源并连接到非计量的Wi-Fi时才会自愿参与优化。此外，我们预计每个客户端每天只会参与少量更新轮次。另一方面，由于任何单个设备上的数据集相对于总数据集大小较小，现代智能手机具有相对快速的处理器（包括GPU），对于许多模型类型来说，相对于通信成本，计算成本几乎为零。因此，我们的目标是利用额外的计算来减少训练模型所需的通信轮数。我们可以通过两种主要方式增加计算：1）增加并行度，即在每轮通信之间使用更多独立工作的客户端；2）增加每个客户端的计算量，即每个客户端在每轮通信之间执行更复杂的计算，而不是执行简单的梯度计算。我们研究了这两种方法，但我们实现的加速主要是由于在每个客户端增加了更多的计算，一旦达到了客户并行的最低水平。

举例说明
假设我们有 3 个客户端 ( K = 3 )，全局数据集有 9 个数据点 ( n = 9 )，各客户端的数据点数量如下：

  • 客户端 1： ( n_1 = 3 )
  • 客户端 2： ( n_2 = 3 )
  • 客户端 3： ( n_3 = 3 )

数据点的损失函数如下：

  • ( f_1(w), f_2(w), f_3(w) ) 在客户端 1
  • ( f_4(w), f_5(w), f_6(w) ) 在客户端 2
  • ( f_7(w), f_8(w), f_9(w) ) 在客户端 3

客户端的本地目标函数 ( F_k(w) ) 分别为：

  • 客户端 1： ( F_1(w) = \frac{1}{3} (f_1(w) + f_2(w) + f_3(w)) )
  • 客户端 2： ( F_2(w) = \frac{1}{3} (f_4(w) + f_5(w) + f_6(w)) )
  • 客户端 3： ( F_3(w) = \frac{1}{3} (f_7(w) + f_8(w) + f_9(w)) )

因此，全局目标函数 ( f(w) ) 可以重写为各客户端本地目标函数的加权平均：

[
f(w) = \frac{n_1}{n} F_1(w) + \frac{n_2}{n} F_2(w) + \frac{n_3}{n} F_3(w)
]

由于每个客户端的数据点数量 ( n_k = 3 )，全局数据点数量 ( n = 9 )，所以权重 ( \frac{n_k}{n} = \frac{3}{9} = \frac{1}{3} )。

最终，全局目标函数重写为：

[
f(w) = \frac{1}{3} F_1(w) + \frac{1}{3} F_2(w) + \frac{1}{3} F_3(w)
]

具体地：

[
f(w) = \frac{1}{3} \left( \frac{1}{3} (f_1(w) + f_2(w) + f_3(w)) \right) + \frac{1}{3} \left( \frac{1}{3} (f_4(w) + f_5(w) + f_6(w)) \right) + \frac{1}{3} \left( \frac{1}{3} (f_7(w) + f_8(w) + f_9(w)) \right)
]

简化后得：

[
f(w) = \frac{1}{9} \sum_{i=1}^9 f_i(w)
]

这与原始的全局目标函数 ( \frac{1}{9} \sum_{i=1}^9 f_i(w) ) 一致。
