标题: CipherGPT: Secure Two-Party GPT Inference

作者: Xiaoyang Hou, Jian Liu, Jingyu Li, Yuhan Li, Wen-jie Lu, Cheng Hong, Kui Ren

作者单位: Zhejiang University, Ant Group

摘要: 文章提出了CipherGPT，这是首个针对GPT模型的两方安全推断框架。该框架通过一系列创新的协议，实现了高效和安全的GPT推断，解决了现有安全推断方案在处理大型GPT模型时的挑战。

关键词: ChatGPT, Secure Inference, Privacy, Two-Party Computation, Matrix Multiplication, GELU, Top-K Sampling.

Abstract

ChatGPT 被认为是人工智能领域的一次重大革命，但它也引发了关于用户隐私的严重担忧，因为用户提交的数据可能包含敏感信息。现有的安全推断解决方案在支持类似GPT的模型时面临显著挑战，这是由于模型参数数量庞大和激活函数复杂所致。

在本文中，我们开发了CipherGPT，这是首个用于安全两方GPT推断的框架，该框架基于一系列创新的协议。首先，我们提出了一种为GPT推断定制的安全矩阵乘法，相较于目前最先进的技术（SOTA），实现了最高6.2倍的速度提升和4.1倍的带宽减少。我们还提出了一种新的协议，用于安全地计算GELU激活函数，在运行时间、通信量和精度上分别超过SOTA 1.8倍、2.5倍和7.4倍。此外，我们还提出了第一个用于安全top-k采样的协议。

我们为CipherGPT提供了完整的实现和全面的基准测试。特别地，我们测量了每个操作的运行时间和通信量，以及它们各自的比例。我们相信，这可以作为未来该领域研究的参考。

Introduction

ChatGPT 是一种大型语言模型 (LLM)，基于具有突破性的生成预训练变换器 (GPT) 架构 [41] 构建，被认为是人工智能领域的一次重大革命。凭借庞大的知识库和出色的语言能力，ChatGPT 在多个任务中表现出色，包括问答、文章润色、建议提供以及对话交流。它还可以作为虚拟助手，有效支持诸如客户支持、信息检索和语言翻译等应用。

OpenAI 将 ChatGPT 开发为一个在线推断服务，并提供了一个远程 API 供开发者使用。用户可以通过提交提示或消息来方便地使用 GPT 推断服务。然而，这种服务模式不可避免地会将用户隐私置于风险之中，因为用户提交的数据可能包含敏感信息。这种隐私问题可能会限制 GPT 在某些数据保密性至关重要的场景中的部署。

安全推断 [22], [34], [31], [37], [44], [43], [29], [27], [40], [30] 是一种两方加密协议，旨在使推断过程中的服务器（S）无法获取关于客户端（C）输入的任何信息，而客户端（C）除了推断结果外对模型一无所知。其基本思路是通过同态加密和秘密共享等定制的加密技术，让 S 和 C 在加密的输入上运行加密的模型。通常会引入一个预处理阶段来准备一些昂贵的且与输入无关的工作，以便在线阶段能够高效完成。

然而，现有的安全推断协议在支持 GPT 方面能力有限。诸如 GPT-2 等生成型大型语言模型（LLM），包含12个变换器，涉及大量高维矩阵乘法和复杂的数学函数如 GELU。在生成任务的情况下，LLM 需要通过逐词生成句子并使用随机选择机制来确保创意性和多样性。另一方面，Iron [27] 仅适用于基于变换器的非生成任务模型，如 BERT。Bolt [40] 和 BumbleBee [30] 优化了通信中的矩阵乘法，但引入了更多计算复杂性。最先进的计算 GELU 激活函数的方法 [40], [30] 使用高阶多项式来逼近曲线，难以同时达到精确性和高效性。之前的研究采用混淆电路 (GC) [58] 实现随机采样，这在计算和通信上极为繁重，或者用选择得分最高的词来替代，从而降低了 LLM 的实用性。因此，GPT 的出现确实给安全推断领域带来了新的挑战。

Contribution

在本文中，我们开发了CipherGPT，这是首个用于安全两方GPT推断的框架，基于一系列新颖的协议。

基于VOLE的矩阵乘法：GPT接受长句子作为输入，并以自回归的方式生成响应词。具体来说，在生成一个响应词后，该词被添加到输入句子中，新的句子成为模型的输入，继续生成下一个响应词。每个响应词的生成都需要进行一次模型推断，这涉及到在每一层的矩阵乘法（简称MatrixMul）。在预处理阶段的每一层，我们将每个响应词的MatrixMul组合成一个单一的不平衡矩阵乘法，并使用sVOLE进行处理。

矢量盲线性评估VOLE [8], [10], [57], [9] 用于生成如 (w = ux + v) 的相关性，其中输入为 (x) 的发送方学习到长度为 (n) 的向量 (w)，而接收方学习到长度为 (n) 的 (u) 和 (v)。子域VOLE（sVOLE）[10] 是VOLE的泛化版本；理想情况下，sVOLE可以在保持与单个VOLE实例运行成本相当的情况下，完成相当于 (k) 个普通VOLE实例的任务。当 (n > k) 时，sVOLE 更具成本效益，因此特别适用于计算不平衡的矩阵乘法。

基于样条的GELU：GPT使用GELU作为其激活函数，可以表示为：

[
\text{GELU}(x) = 0.5x(1 + \text{Tanh}(\sqrt{2/\pi}(x + 0.044715x^3)))
]

其中 (\text{Tanh}(x) = 2\text{Sigmoid}(2x) - 1) 和 (\text{Sigmoid}(x) = \frac{1}{1+e^{-x}})。为了安全地计算GELU，SIRNN [43] 和 Iron [27] 使用查找表（LUT）[18] 来逼近 (e^{-x}) 以及另一个查找表来逼近倒数。这个多步骤过程还需要在每个步骤中扩展或截断比特宽度，以平衡精度和效率。最先进的方法（SOTA）[40], [30] 将GELU分成几个部分，然后使用高阶多项式来逼近每个部分的曲线。最后 [40], [30] 使用比较和多路复用器来找到正确的部分并选择相应的输出。

与此不同的是，我们仅使用线性函数来逼近曲线部分。为此，我们将GELU的曲线部分划分为几个区间，并在每个区间内使用线性函数（(y = ax + d)）来逼近曲线。这种基于样条的逼近最初由Liu等人[34]提出，他们使用混淆电路 [58] 来找到x所在的区间并计算相应的线性函数。我们通过利用LUT [18]来找到区间，并以秘密共享的方式计算相应的线性函数，大大提高了其性能。

与基于LUT的方法 [43], [27]相比，我们使用更少的LUT并节省了大量的加密原语。SIRNN [43] 和 Iron [27] 使用LUT找到初始值并执行迭代来逼近非线性函数，而我们使用一个LUT找到线性函数并执行乘法-截断来计算相应的值。与基于高阶多项式的方法 [40], [30], [20]（SOTA）相比，我们节省了大量的乘法-截断和比较操作。BOLT [40]和BumbleBee中的乘法-截断次数取决于高阶多项式的度数，每个分段需要一次比较和一个多路复用器。我们始终使用一个LUT（与区间数量无关）和一次乘法-截断来计算线性函数。此外，我们的方法具有更高的精度，避免了多步骤方法 [27], [43] 中固有的误差累积，以及高阶多项式逼近 [40], [30], [20] 中的精度损失。

基于混洗的Top-K选择：从长度为n的秘密共享向量中选择Top-K元素的一种直接方法是安全排序向量，这通常是通过安全执行数据无关的排序算法（如Bitonic排序网络 [28]）来实现的。

我们的第一个见解是基于[3]的想法实现安全排序：首先对输入元素进行安全混洗；然后应用基于输入保护的比较排序协议（例如带安全比较的快速排序）来将混洗后的元素排列成有序状态。注意，在排序过程中获得的比较结果不会泄露关于原始元素的任何信息，因为这些元素已经被混洗。

我们的第二个见解是，如果使用快速排序，则无需对整个向量进行排序。相反，我们可以利用快速排序算法的修改版本。通常，快速排序随机从向量中选择一个元素作为枢轴，并将其与其他元素进行比较。根据比较结果，将向量分为两部分：小于枢轴的元素和大于枢轴的元素。快速排序算法递归地对这两个部分进行操作。在我们的案例中，我们只需递归处理包含Top-K最大元素的部分，这将比较次数从 (O(n \log n)) 减少到 (O(n))。

安全采样：我们还解决了基于秘密共享概率从向量中安全采样元素的问题。具体来说，给定一个包含K个元素的向量，每个元素都与一个秘密共享概率 (p_i) 相关联，第 (j) 个元素被选中的概率为 (p_j)。我们的协议只需要 (K - 1) 次安全比较和 (K) 个多路复用器。我们提到，我们的案例与安全采样MPC差分隐私 [54] 的离散高斯噪声不同。他们的直觉是从已知分布中进行安全采样，而我们则是基于秘密共享概率从向量中安全采样一个元素。据我们所知，我们的研究代表了对安全采样的首次探索。

我们总结我们的贡献如下：

  • 一种定制的GPT安全矩阵乘法，与SOTA相比，实现了高达6.2倍的加速和4.1倍的带宽减少（详见第III节）；
  • 一种新的安全计算GELU的协议，在运行时间、通信量和精度上分别超过SOTA 1.8倍、2.5倍和7.4倍（详见第IV节）；
  • 一种创新的Top-K采样解决方案：选择Top-K概率并根据选定的概率采样一个元素（详见第V和VI节）；
  • 首个用于安全两方GPT推断的框架（详见第VII节），以及一个全面的基准测试，可作为该研究方向的参考（详见第VIII节）。

Preliminaries

在本节中，我们将介绍理解本文所需的基础知识。表I总结了本文中常用的符号和术语。

A.安全推断和威胁模型

安全推断是一种两方密码协议，用于在客户端 (C) 和服务器 (S) 之间进行模型推断。该协议确保 (C) 只能获取有关模型架构和推断结果的信息，而 (S) 的模型的所有其他细节都保持隐藏。同样，(S) 只能知道 (C) 的输入长度和推断输出的长度，而不了解其中的确切内容。在GPT推断的背景下，(C) 的输入是一个提示（prompt），而 (S) 的输入是一个经过充分训练的模型，该模型包含多个transformer解码器以及一个vec2word层。关于GPT的更多信息，请参见第VII节。

我们假设 (C) 或 (S) 都可能是“半诚实”的对手，这意味着它们会遵循协议的规范，但在协议执行过程中会尽可能多地收集信息。我们假设对手的计算能力是有界的，并使用 (\lambda) 表示计算安全参数。

子域矢量盲线性评估（Subfield Vector Oblivious Linear Evaluation, VOLE）：VOLE是一种两方功能，它从发送方获取一个标量 (x \in \mathbb{F}_p)，并生成一个VOLE相关性：

[
w = ux + v,
]

使得接收方学到 ((u, v) \in_R \mathbb{F}_p^n \times \mathbb{F}_p^n)，而发送方学到 (w \in_R \mathbb{F}_p^n)。VOLE 通常基于噪声学习奇偶性（LPN）和可打孔PRF构建，使得其复杂性几乎独立于 (n)。

子域VOLE（sVOLE）：这是VOLE的一个推广，具有 (u \in_R \mathbb{F}_p^n)，(x \in \mathbb{F}_q)，(w, v \in_R \mathbb{F}_q^n)，且 (q = p^m)。注意，sVOLE 实现了与运行 (m) 个常规VOLE实例相同的任务，但成本更低。

VOLE 和 sVOLE 最初是在有限域上提出的。Baum等人 [6] 提出了在有限环（如 (\mathbb{Z}_{2^l})）上工作的方式。

同态加密（Homomorphic Encryption, FHE）：全同态加密是一种加密方案，允许对加密数据进行任意操作 [21]。在实践中，它通常以分层方式使用：操作只能进行有限次数，否则密文无法解密。在大多数FHE密码系统中 [12], [11], [19], [15]，明文被编码为商环 (\mathbb{Z}_p[x]/(x^N + 1)) 中的多项式，其中 (N) 是2的幂，(p) 是明文模数。然后，明文多项式被加密成包含两个多项式的密文，这些多项式位于 (\mathbb{Z}_q[x]/(x^N + 1)) 中，其中 (q) 是密文模数，决定了安全级别以及可以执行加法和乘法操作的次数。

批量盲线性评估（Batch Oblivious Linear Evaluation, BOLE） 是一种两方功能，它从发送方接收 (x \in \mathbb{F}_p^n) 和从接收方接收 (y \in \mathbb{F}_p^n)，生成一个 BOLE 相关性：

[
v_i + w_i = x_i \cdot y_i \quad (2)
]
