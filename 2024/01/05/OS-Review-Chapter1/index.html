Chapter01

OS:

  • 负责协调硬件、软件等计算机资源的工作
  • 为上层用户、应用程序提供简单易用的服务
  • 一种系统软件


OS的功能：

  • 命令接口：user
    • 联机命令接口(分时/实时): 一句
    • 脱机命令接口(批处理系统)： 一堆
  • 程序接口：user -> program , 一系列system call组成


OS的特征：

  • 并发
  • 共享
    • 互斥
    • 同时
  • 虚拟
    • 空分
    • 时分
  • 异步


OS发展和分类:

  • 手工
  • 批处理(脱机)：
    • 单道批处理：脱机输入输出
    • 多道批处理：多道程序并发，提高系统吞吐量和利用率，但无人机交互
  • 分时(联机)：时间片为单位，人机交互，不能处理紧急问题
  • 实时(联机)：
    • 软：绝对严格时间内完成处理
    • 硬：允许偶尔违反时间规定
  • 网络
  • 分布式
  • 个人
操作系统类型 优点 缺点
批处理操作系统 用户脱机使用计算机；作业成批处理；系统内多道程序并发执行 交互能力差
分时操作系统 多用户同时使用；人机交互性强；每个用户具有使用计算机的独占性；系统响应及时 不能处理紧急问题
实时操作系统 对控制对象及时反应；可靠性高；响应及时 资源利用率低


OS运行机制和体系结构：

  • 运行机制：
    • 两种指令
      • 特权指令：eg：内存清零
      • 非特权指令： eg：运算指令
    • 两种处理器状态
      • 核心态：特权+非特权
      • 用户态：非特权
    • 两种程序
      • 内核程序
      • 应用程序
  • 内核：
    • 时钟
    • 中断
    • 原语：特殊的程序，处于操作系统最底层，最接近硬件的部分
      • 特点：一气呵成不能中断，运行时间短
    • 资源管理
      • 进程
      • 存储器
      • 设备
  • 体系结构：
    • 大内核：主要功能模块作为系统内核，运行在核心态
      • 高性能
      • 难以维护
    • 微内核：基本功能保留在内核
      • 方便维护
      • 需要频繁在核心态和用户态之间切换


中断与异常：

  • 中断：为了处理多道程序并发执行而引入的一种技术，是CPU从用户态进入核心态的唯一途径。发生中断就意味着需要操作系统介入开展管理工作，CPU立即进入核心态。
  • 分类：
    • 内中断(异常，例外，陷入)：来源CPU内部
      • 自愿中断：eg：trap指令
      • 强迫中断：eg：软硬件故障
    • 外中断(中断)：来源CPU外部
      • 外设请求：eg：I/O操作完成
      • 人工干预 eg：用户强制终止进程
  • 外中断的处理过程：每条指令执行结束后，CPU检查是否有外部中断。若有外部中断，则需要保护被中断进程的CPU环境。根据中断信号类型转入相应的中断处理程序。恢复原进程的CPU环境并退出中断，返回原进程继续往下执行。



系统调用：

  • 定义：操作系统给应用程序的接口
  • 作用：应用程序通过系统调用来请求获得操作系统服务，让处理器从用户态进入核心态。
  • 分类：
    • 设备管理
    • 文件管理
    • 进程控制
    • 进程通信
    • 内存管理
  • 与库函数的区别：
  • 系统调用背后的过程：
    • 传递系统调用参数
    • 执行trap指令(用户态)
    • 执行系统调用相对服务程序(核心态)
    • 返回用户程序

CPU与I/O设备：

无论内置在主板上的接口or集成在设备上的接口，除三类寄存器，还有对应控制电路。正是控制电路，CPU才能通过向这个接口电路板传输信号，控制实际硬件。

数据寄存器（Data Register）: CPU向I/O设备写入需要传输的数据，比如要打印“Time”，就要先发送一个“T”到对应I/O设备。

命令寄存器（Command Register）:CPU发送一个命令，告诉打印机，要进行打印。这时，打印机里的控制电路会：

  • 设置状态寄存器里面的状态，把状态设置成not-ready
  • 实际操作打印机进行打印

状态寄存器（Status Register）:告诉CPU，现在设备已经在工作了，这时，CPU你再发送数据或者命令过来，都是没有用的。直到前面的动作已经完成，状态寄存器重新变成了ready状态，我们的CPU才能发送下一个字符和命令。

分页与缺页

分页是一种将内存划分为块并进行管理的技术，使得操作系统可以更有效地使用内存。

缺页是当程序尝试访问一个未映射到物理内存的页框时发生的情况，操作系统需要采取措施来加载所需的数据并更新映射关系。

例子：

假设你有一台计算机，其物理内存大小为 4KB，而每个页面的大小为 1KB。这意味着物理内存可以被划分为 4 个页面（Page 0, Page 1, Page 2, Page 3）。同时，假设这台计算机使用了虚拟内存技术，虚拟内存的大小为 8KB，因此可以被划分为 8 个页框（Page Frame 0 至 Page Frame 7）。

例子1:

  1. 操作系统如何管理内存: 当程序运行时，它们在虚拟内存中存储数据和代码。操作系统的任务是将这些虚拟页映射到物理内存的实际页中。例如，如果程序的第一个页框（Page Frame 0）被映射到物理内存的第三页（Page 2），操作系统会在页表中记录这种映射关系。

  2. 访问内存: 当程序尝试访问位于 Page Frame 0 的数据时，操作系统查看页表，确定这些数据实际上位于物理内存的 Page 2，然后将访问重定向到正确的物理位置。

例子2:

假设程序尝试访问位于 Page Frame 5 的数据，但该页框尚未映射到物理内存的任何页中。

  1. 触发缺页: 操作系统检测到 Page Frame 5 并没有映射到物理内存的任何页，因此产生一个缺页。

  2. 处理缺页: 操作系统选择一个物理页（比如 Page 1），如果该页中有数据，操作系统可能先将其保存到硬盘上（如果这些数据被修改过）。然后，它从硬盘上的交换空间中读取原本应在 Page Frame 5 的数据，将其加载到物理内存的 Page 1。

  3. 更新页表和恢复执行: 一旦数据被加载到物理内存中，操作系统更新页表，记录 Page Frame 5 现在映射到物理内存的 Page 1。之后，程序可以继续执行，访问它需要的数据。

访管中断

访管中断（Supervisor Call Interrupt，或称为特权指令中断）通常发生在以下情况：

  1. 非特权模式下执行特权指令：当一个运行在用户模式（非特权模式）的程序试图执行仅限于内核模式（特权模式）的指令时，会引发访管中断。这是一种安全机制，用于防止非授权的代码修改系统级的设置或访问关键资源。

  2. 系统调用：用户程序经常需要执行一些只能由操作系统内核执行的操作，如文件操作、网络通信等。这些操作通过系统调用来实现。当程序执行系统调用时，实际上是触发了访管中断，这使得CPU从用户模式切换到内核模式，以便安全地执行那些需要更高权限的操作。

  3. 访问受保护资源：如果用户模式下的程序尝试直接访问受操作系统保护的资源，如直接访问硬件设备或修改关键的系统数据结构，操作系统会通过访管中断来拦截这些尝试。
