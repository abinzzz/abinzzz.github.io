Chapter02


1.1 进程定义，组成，组织方式

进程定义：进程实体的运行过程，是系统进行资源分配和调度的一个独立单位

与程序的区别：

  • 程序：静态
  • 进程：动态

进行进程切换的时候需要把当前的运行状态记录下来保存在PCB中

进程组成：

  • （1）PCB(进程的管理者所需的数据都放在PCB中)：
    • （1-1）进程描述信息：
      • 进程标识符PID
      • 用户标识符UID
    • （1-2）进程控制和管理信息：
      • 进程当前状态
      • 进程优先级
    • （1-3）资源分配清单
      • 程序段指针
      • 数据段指针
      • 键盘
      • 鼠标
    • （1-4）处理机相关信息：各种寄存器值
  • （2）程序段：代码
  • （3）数据段：数据

组织方式：

  • 链接方式
  • 索引方式


进程的特征：

  • 动态：动态产生，变化，消亡
  • 并发：
  • 独立：
  • 异步：
  • 结构：PCB+程序段+数据段


1.2 进程状态与转换

状态:

  • 运行态
  • 就绪态
  • 阻塞态
  • 创建态: 进程正在被创建，操作系统为进程分配资源、初始化PCB
  • 终止态: 进程正在从系统中撤销，操作系统回收进程拥有的资源、撤销PCB


1.3 进程控制

定义：进程状态的转换，使用原语实现

原语：

  • 使用开关中断来实现(核心态下执行的特权指令)
  • 一气呵成

显然，关/开中断指令的权限非常大，必然是只允许在核心态下执行的特权指令


进程控制相关原语：

  • （1）过程：
    • 更新PCB中的信息：修改进程状态标志，剥夺CPU使用权并保存环境/恢复其运行环境
    • 将PCB插入合适队列
    • 分配/回收资源
  • （2）分类：
    • 创建原语
    • 切换原语
    • 撤销原语
    • 阻塞原语
    • 唤醒原语

1.4 进程通信

进程通信：指进程之间的信息交换

各进程拥有的内存地址空间相互独立，一个进程不能直接访问另一个进程的地址空间

分类：

  • （1）共享存储：互斥访问共享空间
    • 基于数据结构 low
    • 基于存储区 high
  • （2）消息传递：传递结构化信息 （消息头+消息体），两个原语 (发送消息 + 接受消息)
    • 直接通信：放到对方消息队列中
    • 间接通信：先发到中间体
  • （3）管道通信: 设置一个特殊共享文件(管道),其实就是一个缓存区；一个管道智能实现半双工通信；各进程要互斥访问通道；写满不能写，读空不能读；读进程最多只能有一个。


1.5 线程和多线程模型

线程定义：轻量级进程

特性 传统进程机制 引入线程后的机制
资源分配与调度 进程是资源分配和处理机调度的基本单位。 进程是资源分配的基本单位，线程是调度的基本单位。
并发性 只能实现进程间的并发。 支持线程间的并发，提高了并发度。
系统开销 进程间并发需要切换进程运行环境，系统开销较大。 线程间并发，尤其是同一进程内的线程切换，不需要切换整个进程环境，因此系统开销较小。

线程的重要属性：

  • 线程是处理机调度的单位，进程是资源分配的单位
  • 多CPU，各线程可以占有不同的CPU
  • 每个线程有TID和TCB
  • 状态：就绪，阻塞，与你写那个
  • 线程几乎不拥有系统资源
  • 同一进程不同线程间共享进程资源
  • 线程通信无需系统干扰
  • 切换线程比切换进程开销小

线程实现方式：

  • 用户级线程：所有的线程管理工作都由应用程序负责（包括线程切换）
  • 内核级线程：内核级线程的管理工作由操作系统内核完成。
  • 组合方式


2.1 处理机调度的概念层次

调度类型 要做什么 调度发生在 发生频率 对进程状态的影响
高级调度（作业调度） 按照某种规则，从后备队列中选择合适的作业将其调入内存，并为其创建进程 外存 -> 内存（面向作业） 最低 无 -> 创建态 -> 就绪态
中级调度（内存调度） 按照某种规则，从挂起队列中选择合适的进程将其数据调回内存 外存 -> 内存（面向进程） 中等 挂起态 -> 就绪态（阻塞挂起 -> 阻塞态）
低级调度(进程调度) 按照某种规则，从就绪队列中选择一个进程为其分配CPU 内存 -> CPU 最高 就绪态 -> 运行态


挂起与阻塞的区别：

  • 挂起是进程映像在外存
  • 阻塞还在内存

2.2 进程调度的时机, 切换过程, 方式

进程调度： 就绪态 -> 运行态

什么时候需要调度：

  • 主动放弃
    • 正常终止
    • 发生异常终止
    • 主动请求阻塞
  • 被迫中断
    • 时间片用完
    • 更加紧急的事情
    • 更高优先级的进程进入

什么时候不可以调度：

  • 处理中断时
  • OS内核程序临界区
  • 原子操作过程

关于进程在操作系统内核程序临界区中不能进行调度与切换(普通临界区可以进行调度和切换):

  • 临界资源：一个时间段内只允许一个进程使用的资源。各进程需要互斥地访问临界资源。
  • 临界区：访问临界资源的那段代码。


进程调度方式：

  • 非抢占式
  • 抢占式

狭义进程调度与进程切换：

  • 狭义的进程调度： 指的是从就绪队列中选中一个要运行的进程。（这个进程可以是刚刚被暂停执行的进程，也可能是另一个进程，后一种情况就需要进程切换）
  • 进程切"换"：重点在换，是指一个进程让出处理机，由另一个进程占用处理机的过程。

举例说明：


广义的进程调度 = 选择一个进程 + 进程切换


进程切换的过程主要完成了：

  • 对原来运行进程各种数据的保存
  • 对新的进程各种数据的恢复 (程序计数器、程序状态字、各种数据寄存器等处理机现场信息，这些信息一般保存在进程控制块)

注意：进程切换是有代价的，因此如果过于频繁的进行进程调度、切换，必然会使整个系统的效率降低，使系统大部分时间都花在了进程切换上，而真正用于执行进程的时间减少
