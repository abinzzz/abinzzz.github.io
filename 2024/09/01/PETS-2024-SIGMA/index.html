标题: Sigma: Secure GPT Inference with Function Secret Sharing

作者:

  • Kanav Gupta（University of Maryland, College Park）
  • Neha Jawalkar（Indian Institute of Science）
  • Ananta Mukherjee（Microsoft Research）
  • Nishanth Chandran（Microsoft Research）
  • Divya Gupta（Microsoft Research）
  • Ashish Panwar（Microsoft Research）
  • Rahul Sharma（Microsoft Research）

关键词:

  • Function Secret Sharing
  • MPC (Multiparty Computation)
  • 安全推理 (Secure Inference)
  • 变压器 (Transformers)
  • GPT

Conference：PETS 2024

Abstract

安全的二方计算（2PC）能够在推理过程中同时保护专有的机器学习（ML）模型和敏感输入数据的安全。然而，现有的安全推理解决方案存在高延迟和通信开销的问题，尤其是在处理变压器模型时。函数秘密共享（FSS）是一种新兴的模式，通过预处理阶段来实现高效的2PC协议。我们提出了Sigma，这是第一个基于FSS的变压器安全推理的端到端系统。通过为复杂的机器学习功能（如Softmax、GeLU和SiLU）构建新的基于FSS的协议，并在GPU上加速其计算，Sigma将变压器安全推理的延迟相比当前最先进的技术减少了12到19倍。我们展示了第一个生成式预训练变压器（GPT）模型的安全推理。具体来说，Sigma在38秒内执行了Meta的Llama2模型（在HuggingFace上可用，拥有130亿参数）的推理，并在1.5秒内执行了GPT2模型的推理。

Introduction

最近，提供预测即服务（PaaS）的软件公司数量激增。这些公司使用其专有数据训练或微调机器学习模型，然后通过部署收费服务来货币化这些模型，这些服务接收客户数据作为输入并提供推理结果。这类服务的客户（理所当然地）对其输入的隐私感到担忧。例如，有关OpenAI提供的ChatGPT服务的多个隐私问题已经被报道【1, 3, 6】。当PaaS被部署在浏览数据、金融、医疗等敏感数据领域时，这个问题会变得更加严重。在这种情况下，模型提供者本身可能也不愿意以明文方式访问用户输入，因为数据泄露或内部员工违规可能导致法律责任。安全推理领域旨在解决这些问题。使用安全推理的PaaS提供了形式化的安全保证，即在推理过程中，客户端除了推理输出外不会了解模型的任何信息，而模型提供者不会了解客户端的输入信息。虽然安全的二方计算（2PC）【88】的密码学技术在理论上可以解决安全推理问题，但其性能开销难以承受。通过一系列致力于减少性能开销的工作【16, 24, 43, 44, 47, 48, 57, 59, 66, 68, 71, 81, 85】，安全推理系统现在变得越来越实用。Mann等人对安全推理进行了综述【55】。

近年来，基于2PC的安全推理应用范围从拥有数千参数的模型【15, 44, 54, 57–59, 63, 67, 69, 71, 81】，扩展到拥有数百万参数的模型【26, 37, 40, 43, 48, 68, 82, 85】，再到拥有数亿参数的BERT模型【12, 25, 38, 47, 51】。在本文中，我们进一步推进这一方向，提供了对具有数十亿参数的生成式预训练变压器（GPT）模型的安全推理。

基于变压器的生成语言模型由于在各种自然语言任务（如问答、摘要、语言翻译、代码生成【20, 21, 76】）中的出色表现，最近得到了显著关注。除了确保模型和输入隐私之外，这类模型的安全推理还打开了其他有趣的场景，如“提示隐私”。AI公司花费大量精力构建能够产生良好推理结果的提示，并希望将这些提示保密。安全推理允许持有专有提示的公司和持有敏感数据的客户在不泄露各自输入的情况下，从公共语言模型生成推理结果。然而，当前最先进的安全推理系统在变压器上的表现并不理想。

我们认为，安全机器学习推理系统必须满足以下要求：（1）准确性——即安全推理下的准确性应与明文推理相匹配；（2）安全性——即系统应提供标准的2PC安全性；（3）效率——即安全推理的延迟和通信开销应较低；（4）可扩展性——即系统必须能够扩展到拥有数十亿参数的模型。我们展示了现有系统未能满足这些要求中的一个或多个。

现有的安全变压器推理系统包括THE-X【25】、Iron【38】和CrypTen【47, 51, 84】（我们在第8节讨论了其他工作）。THE-X通过将基于初等函数（如𝑒^𝑥）的复杂非线性替换为简单非线性（如max(𝑥, 0)），牺牲了准确性，同时通过暴露中间值牺牲了安全性。Iron虽然保持了准确性和安全性，但通信开销巨大，即使对于BERT模型，也需要超过100GB的通信。虽然CrypTen利用GPU加速和预处理来提高效率，但其在线推理的延迟和通信开销仍然很大。此外，它未能提供标准的2PC安全性，因为它使用了不安全的局部截断。此外，由于GPU内存溢出，它无法扩展到更大的模型。

Premilinaries

2.1 符号

令 ( \lambda ) 为计算安全参数，( N = 2^n ) 且 ( L = 2^\ell )。令 ( \mathbb{R} ) 表示实数集合，( U_{2^n} ) 表示 ( n ) 位无符号整数的集合。我们使用标准的二进制补码表示法来表示 ( U_N ) 中的有符号值。对于 ( x \in U_N )，( \text{int}_n(x) ) 和 ( \text{uint}_n(x) ) 分别表示其在整数集合 ( \mathbb{Z} ) 中对应的有符号和无符号整数。我们使用黑体字来表示数组，其第 ( i ) 个元素（从 0 开始计数）使用相同的符号并采用正常字体，后跟 ([i])，例如，(\mathbf{a} = {a[0], a[1], a[2], \ldots })。

2.1.1 定点表示

定点表示（Fixed-point representation）由位宽 ( n ) 和精度 ( f ) 参数化，将实数 ( r \in \mathbb{R} ) 编码为 ( n ) 位整数 ( x \in U_N )，使得 ( x = \left\lfloor r \cdot 2^f \right\rfloor \mod N )。相反，具有精度 ( f ) 的 ( n ) 位定点数 ( x ) 解码为实数 (\frac{\text{int}_n(x)}{2^f})。

2.1.2 操作符

对于一个谓词 ( b )，( 1{b} \in {0, 1} ) 在 ( b ) 为真时返回 1，否则返回 0。对于 ( n < \ell ) 且 ( x \in U_N )，( \text{extend}_{n, \ell}(x) ) 在左侧填充 ( (\ell - n) ) 个 0。对于 ( x \in U_N )，( \text{MSB}_n(x) \in {0, 1} ) 表示 ( x ) 的最高有效位。

2.1.3 秘密共享

对于 ( x \in U_N )，秘密共享生成随机共享 ( x_0, x_1 \in U_N )，使得 ( x = x_0 + x_1 \mod N )，并记为 ( \text{share}(x) )。当 ( x_0 ) 由 ( P_0 ) 持有且 ( x_1 ) 由 ( P_1 ) 持有时，我们用 ( \text{reconstruct}(x_b) ) 表示交换共享并将它们相加以重建底层值的过程，其中 ( b \in {0, 1} )。

2.2 威胁模型

本研究采用的是预处理模型中的标准两方计算（2PC），这一模型在安全机器学习的背景下也得到了广泛关注。具体来说，系统中有两个参与方 (P_0) 和 (P_1)，它们分别持有输入 (x_0) 和 (x_1)，希望计算一个公共函数 (y = f(x_0, x_1))，而不向对方透露任何超过函数输出 (y) 的信息。在与函数输入 (x_0) 和 (x_1) 无关的预处理阶段中，生成了相关的随机性，并将其提供给 (P_0) 和 (P_1)。这些随机性可以通过多种方式生成：可信的第三方、通用的2PC协议，或者通过专门的2PC协议。在本文中，我们考虑了第一种方法。我们的所有协议都满足基于模拟的标准安全概念，并提供对半诚实静态概率多项式时间（PPT）攻击者的安全性，无论攻击者破坏的是 (P_0) 还是 (P_1)。

2.3 函数秘密共享

函数秘密共享（Function Secret Sharing, FSS）方案是一对算法（Gen, Eval）。Gen 将一个函数 (g) 拆分为两个函数共享（(g_0, g_1)），而 Eval 接受输入 (b \in {0, 1})、函数共享 (g_b) 和输入 (x)，并返回 (g_b(x))。FSS 方案的正确性要求对所有 (x)，(g_0(x) + g_1(x) = g(x))。安全性要求是，每个函数共享 (g_b) 都能隐藏函数 (g)。

定义1 (FSS: 语法): 一个两方 FSS 方案是一对算法（Gen, Eval），满足以下条件：

  • Gen((1^\lambda, \hat{g})) 是一个概率多项式时间（PPT）的密钥生成算法，给定 (1^\lambda) 和 ( \hat{g} \in {0, 1}^* )（函数 (g) 的描述），输出一对密钥（(k_0, k_1)）。我们假设 ( \hat{g} ) 明确包含输入和输出组 (G{\text{in}})、(G{\text{out}}) 的描述。
  • Eval((b, k_b, x)) 是一个多项式时间的评估算法，给定 (b \in {0, 1})（参与方索引）、(k_b)（定义 (g_b: G^{\text{in}} \rightarrow G^{\text{out}}) 的密钥）和 (x \in G^{\text{in}})（(g_b) 的输入），输出 (y_b \in G^{\text{out}})（(g_b(x)) 的值）。

( (k_0, k_1) ) 被称为 FSS 密钥，存储一个 FSS 密钥所需的位数称为密钥大小（key size）。我们在附录B中正式定义了FSS方案的正确性和安全性。
