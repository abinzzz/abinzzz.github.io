1.1 初识文件管理概念和功能

文件：一组有意义的信息/数据集合

文件的属性

文件名：由用户决定，同目录下不许重名
标识符：一个系统内的各文件标识符(唯一)
类型：文件的类型
位置：文件存放的路径(用户)、外存中的地址(系统)
大小
创建时间
上次修改时间
文件所有者信息
保护信息：对文件进行保护的访问控制信息

文件内的组织方式

文件的分类：

  • 无结构文件/流式文件：由一些二进制或字符流组成
  • 有结构文件/记录式文件：由一组相似的记录组成

数据项：文件系统中最基本的数据单位
记录:一组相关数据项的集合

文件间的组织方式

使用目录/文件夹
目录也是一种特殊的有结构文件

文件的基本功能

  • 创建文件：create
  • 删除文件：delete
  • 读文件：read
  • 写文件：write
  • 打开文件：open(read/write之前必须进行)
  • 关闭文件：close(read/write之后必须进行)

文件存放在外存

块/磁盘块/物理块(外存)：文件分配存储单位，每个块相等且包含2的整数幂个地址
文件的逻辑地址：(逻辑块号，块内地址)
文件的物理地址：(物理块号，块内地址)
外存和内存的相同点：由一个个存储单元组成，每个存储单元可以存储一定量的数据且对应一个物理地址

操作系统实现的文件管理功能

文件共享：多个用户共享同一个文件
文件保护：保证不同用户对文件由不同的操作权限


1.2 文件逻辑结构

文件的逻辑结构：用户看来，文件内部的数据应该是如何组织起来
文件的物理结构：操作系统看来，文件的数据是如何存放在外存中的

文件的逻辑结构分类：

  • 无结构文件
  • 有结构文件
    • 顺序文件
    • 索引文件
    • 索引顺序文件

无结构文件

无结构文件又称流式文件(例如.txt文件)，由一系列二进制或字符流组成，没有明显的结构特点，所以不探讨该文件的逻辑结构

有结构文件

有结构文件/记录式文件：一组相似记录组成
记录：若干数据项组成，每条记录又一个数据项可作为关键字(类比database中的primary key)

按照记录的长度进行分类：

  • 定长记录
  • 不定长记录

按照逻辑结构分类：

  • 顺序文件
  • 索引文件
  • 索引顺序文件

有结构文件：顺序文件

顺序文件：文件中的记录一个接一个顺序排序(逻辑上)
链式存储(类比链表)：逻辑上相邻的记录物理上也相邻， 无法随机存取，每次只能从第一个记录开始查找
顺序存储(类比顺序表)：逻辑上相邻但物理上不一定相邻，变长无法随机存取，定长可以随机存取(记录长度为L，第i个记录存放的相对位置为i*L)；串结构无法快速找到关键字对应记录顺序结构(按关键字排序)可以
缺点：增加/删除一个记录困难

有结构文件：索引文件

由于上面只有定长的顺序存储的顺序文件可以实现随机存取，但不定长的却无法解决，所以我们使用索引文件，每条记录对应一个索引，每个索引都是定长的，这样对于不定长的记录，就可以使用定长的顺序存储的顺序文件来进行随机存取啦

思想：记录不是定长，索引是定长的

索引表：定长记录的顺序文件，根据上面内容我们可以知道能快速找到第i个记录对应的索引项

有结构文件：索引顺序文件

对于索引文件，每个记录对应一个索引表项，因此索引表会很大，存储空间利用率太低

索引顺序文件：索引文件+顺序文件，一组记录对应一个索引表项

效率分析：

  • 顺序文件有10000个记录，从头顺序查找，平均需要查找5000个
  • 索引顺序文件10000=100x100，在100个索引表中找到分组，平均查找50次；在分组中顺序查找记录，平均需要查找50次，总的查找次数50+50=100

1.4 文件的物理结构

文件的物理结构：

  • 连续分配
  • 链式分配
    • 隐式链接
    • 显示链接
  • 索引分配

文件块与磁盘块

外存中也会分页，被分为一个个块/物理块/磁盘块(与内存块/页面大小相同)，是磁盘与内存的数据交换单位

文件块：文件的逻辑地址空间被分为一个个的块
文件的逻辑地址：逻辑块号+块内地址

连续分配

连续分配：每个文件在磁盘上占有一组连续的块，支持顺序访问和随机访问
逻辑地址 -> 物理地址：(逻辑块号，块内地址) -> (物理块号，块内地址)
物理块号 = 起始块号 + 逻辑块号

文件名 … 起始块号 长度
aaa … 4 3
bbb … 10 4
… … … …

优点：支持顺序和随机访问，顺序读写最快
缺点：物理上连续分配不方便拓展，存储空间利用率低，会产生碎片

为了解决该缺点，我们提出了链接分配

链接分配

链接分配：采用离散分配的方式，可以将文件
链接分配的分类：

  • 隐式链接
  • 显示链接

隐式链接：

  • 增加了结束块号
  • 除了最后一个磁盘块，每个磁盘块都会保存指向下一个磁盘块的指针(用户可见)
  • 优点：空闲磁盘块充分利用，方便拓展
  • 缺点：只支持顺序访问，不可随机访问
文件名 … 起始块号 结束块号
aaa … 9 8

显示链接：

  • 目录现在只有起始块号
  • 用于链接文件各物理块的指针放在文件分配表FAT中
  • 优点：可顺序访问可随机访问，会产生外部碎片，相比于隐式链接不需要访问磁盘
  • 缺点：FAT需要占空间

目录：

文件名 … 起始块号
aaa … 2
bbb … 4

FAT：一个磁盘仅一张FAT，常驻内存

物理块号(连续存储可隐含) 下一块
0 1
1 -1
2 5
3 -1
4 23
5 0
… …
22 …
23 3

索引分配

索引分配：允许文件分散地在各个磁盘中，每个文件建一张索引表
索引表：记录文件各个逻辑块对应的物理块
索引块：索引表存放的磁盘块
数据块：文件数据存放的磁盘块
索引表 VS FAT: 索引表是一个文件对应一个，FAT是一个磁盘对应一个

优点：支持随机访问，文件拓展也容易实现
缺点：索引表占空间

目录：

文件名 … 索引块
aaa … 7
bbb … 23

文件aaa的索引表：

逻辑块号(顺序存储可隐含) 物理块号
0 2
1 5
2 13
3 9

数据太大，索引表无法装下：

  • 方案1:链接方案
  • 方案2:多层索引
  • 方案3:混和索引

链接方案：多个索引块链接起来存放,低效

多层索引：建立多层索引(类比多级页表)，但各层索引表大小不能超过一个磁盘块

混和索引：多种索引分配方式结合,顶级索引表中包含直接地址索引，一级间接索引，二级间接索引

对比总结

分配方式 必须条件 目录项内容 优点 缺点
顺序分配 为文件分配的必须是连续的磁盘块 起始块号、文件长度 顺序存取速度快, 支持随机访问 会产生碎片，不利于文件拓展
隐式链接 每个盘块中都存有指向下一个盘块的指针 起始块号、结束块号 可解决碎片问题，外存利用率高，文件拓展实现方便 只能顺序访问，不能随机访问。
显式链接 建立一张文件分配表(FAT)，显式记录盘块的先后关系 起始块号 除了拥有隐式链接的优点之外，还可通过查询内存中的FAT实现随机访问 FAT需要占用一定的存储空间
索引分配 为文件数据块建立索引表。记录的是顶级索引块的块号 链接方案记录的是第一个索引块的块号，多层/混合索引记录的是顶级索引块的块号 支持随机访问，易于实现文件的拓展 索引表需占用一定的存储空间。访问数据块前需要先读入索引块。若采用链接方案，查找索引块时可能需要很多次读磁盘操作

1.5 空闲磁盘块的几种算法

磁盘可以分为非空闲磁盘块和空闲磁盘块，上面已经对非空闲磁盘块的管理方式(文件的物理结构)进行了介绍，下面就来介绍对空闲磁盘块的管理方式

空闲磁盘块管理方法的分类：

  • 空闲表法
  • 空闲链表法
    • 空闲盘块链
    • 空闲盘区链
  • 位示图法
  • 成组链接法

存储空间的划分与初始化

磁盘需要分区：C:盘、D:盘、E:盘(文件卷C，D，E)…
存储空间初始化(各个文件卷划分):目录区和文件区

空闲表法

适用：连续分配方式

分配方法：与动态分区分配类似，为一个文件分配连续的存储空间，同样可使用首次适应，最佳适应，最坏适应等…

回收方法：与动态分区分配类似，回收有四种情况：

  • 前后没有相邻空闲区
  • 前后都是空闲区
  • 前面是空闲区
  • 后面是空闲区

空闲链表法

空闲链表法分类：

  • 空闲盘块链：盘块为单位
  • 空闲盘曲链：盘区为单位


空闲盘块链：OS保存链头链尾指针
适用：离散分配的物理结构
分配：申请K个盘块，则从链头开始依次摘K个盘块分配，修改链头指针
回收：挂到链尾，修改链尾指针

空闲盘区链：OS保存链头链尾指针
适用：离散、连续都适用
分配：申请K个盘块，采用首次适应，最佳适应，最坏适应等找到一个符合大小的空闲分区
回收分两种情况：

  • 回收区和空闲盘区相邻，合并
  • 不相邻，回收区单独作为空闲盘区挂到链尾

位示图法

位示图：每个二进制位对应一个盘块,0是空闲，1是已分配
盘块号 = 字长 x 字号 + 位号
b号盘对应的字号=b/字长，位号b%字长
分配：顺序扫描扎到K个相邻/不相邻的0，根据字号、位号推算出盘块号，分配给文件后设置为1
回收：根据回收的盘块号计算字号、位号，设置为0


成组链接法

由于前面的方法不适用于大型文件系统，于是我们使用成组链接法
文件卷的目录区专门有一个磁盘块作为超级块，超级块要被读入内存且内外存中要一致

分配：检查第一个分组是否足够，足够则修改第一个分组中的空闲块数，更新超级块的信息指向新的空闲块组和空闲块
回收：类似链表头插法


1.6 文件的基本操作原理

创建文件

Create系统调用(外存空间大小，文件存放路径，文件名)：

  • 外存中找到文件所需空间(1.5节中的空闲磁盘块算法)
  • 通过路径找到目录文件，目录中创建该文件对应的目录项(包含FCB)

删除文件

Delete系统调用(路径，文件名)

  • 路径找目录文件，目录中找到文件名对应的目录项
  • 回收磁盘块(上一小节)
  • 删除文件对应的目录项

打开文件

open(路径，文件名，操作类型)

  • 找到文件名对应的目录项
  • 目录项复制到内存中的打开文件表中，使用打开文件表的编号来指明要操作的文件

  • 进程打开文件夹

读文件

read(文件，数据，放到内存中的位置)：从读指针指向的外存中将用户指定大小的数据读入用户制定的内存中

写文件

write(文件，数据，放到内存中的位置)：从用户指定的内存中，将指定大小的数据写回到写指针指向的外存


1.7 文件共享(与复制不同)

文件共享：

  • 硬链接(索引结点)
  • 软链接(符号链)

硬链接

索引结点：由于检索文件的时候只需要用到文件名，因此将文件名以外的信息放到索引结点中

索引结点中设置一个链接计数变量count：链接到本索引结点上的用户目录项数

  • count=2:说明有两个用户目录项连接到该索引结点上
  • count>0:有人要用文件
  • count=0:系统删除文件

软链接

Link型文件记录了文件的存放路径(类似于Windows中的快捷方式)

系统访问”ccc“时候，操作系统判断文件”ccc“属于link类型文件，于是根据记录中的路径层层查找，找到"aaa"项于是就找到了文件的索引结点


1.8 文件保护

文件保护分类：

  • 口令保护
  • 加密保护
  • 访问控制

口令保护

为文件设置一个口令，用户请求访问该文件必须提供口令
口令存放位置：FCB

优点：时空开销小
缺点：口令放在系统内部，不够安全

加密保护

访问文件时候，需要提供正确的密码仓能看到文件的正确解密

优点：保密性强
缺点：编码/译码花费时间

访问控制

FCB/索引结点中增加一个访问控制列表(ACL),该表记录用户可以对文件执行哪些操作

举例如下：

完全控制 执行 修改 读取 写入
系统管理员 1 1 1 1 1
文件主 0 1 1 1 1
文件主的伙伴 0 1 0 0 0
其他用户 0 0 0 0 0

1.9 文件系统的层次结构

  • 用户/应用程序
  • 用户接口：处理系统调用
  • 文件目录系统：根据路径找到对应的FCB或者索引结点，与目录紧密相关
  • 存取控制模块：验证访问权限
  • 逻辑文件系统与文件信息缓冲区：文件记录号->逻辑地址
  • 物理文件系统：逻辑地址->物理地址
  • 辅助分配模块：存储空间管理
  • 设备管理模块：与硬件交互

2.1 磁盘的结构

磁盘、磁道、扇区

磁道：磁盘盘面的一个个圈
扇区/磁盘块：磁道被划分

如何读写数据

需要把磁头移动到想要读写扇区所在的磁道

盘面、柱面

用柱面号/盘面号/扇区号来定位一个磁盘块，磁盘块号可以转换成柱面号/盘面号/扇区号的地址形式

根据地址读取块：

  • 根据柱面号让磁头指向指定柱面
  • 激活指定盘面对应的磁头
  • 既定扇区从磁头划过

磁盘分类

按磁头可移动分类：

  • 活动头磁盘
  • 固定头磁盘

按盘片是否可替换分类：

  • 可换盘磁盘
  • 固定盘磁盘


2.2 磁盘调度算法

磁盘调度算法分类：

  • 先来先服务
  • 最短寻找时间优先
  • 扫描算法
  • 循环扫描算法

一次磁盘读写操作需要的时间

寻找时间TST_STS​:读写数据之前，磁头移动到指定磁道花费的时间，TST_STS​=启动磁头臂时间+磁道书x跨磁道耗时
延迟时间TRT_RTR​:1/(2x磁盘转速)
传输时间TtT_tTt​:磁盘读出到磁盘写入的时间，TtT_tTt​=读写字节数/(磁盘转速x磁道字节数)
总时间TaT_aTa​=上面求和

先来先服务

思想：根据进程请求访问磁盘的先后顺序进行调度
优点：公平
缺点：请求访问的磁盘很分桑，寻道时间长

最短寻找时间优先算法

思想：优先处理于当前磁头最近的磁道
优点：平均寻道时间短
缺点：可能饥饿

扫描算法

上面算法产生饥饿的原因：磁头可能在一个小区域中来回移动
思想：只有磁头移动到最外侧才能往内移动，移动到最内侧才能往外移动
优点：寻道时间短，不会饥饿
缺点：只有到达最边上才能改变磁头移动方向，其实可以提前改变移动方向；各个位置相应频率不平均

LOOK算法

为了解决上面的缺点(到达最边上才能改变磁头移动方向)，如果磁头移动方向没有别的请求可以立即改变磁头移动方向

循环扫描算法C-SCAN

为了解决相应频率不平均，朝着特定方向移动才会处理磁道访问请求，返回时直接快速移动不处理任何请求
优点：相应频率平均
缺点：到达最边上才能改变磁头移动方向

C-LOOK算法

上面两者的综合

磁盘初始化

扇区的组成：

  • 头(管理扇区所需的数据结构)
  • 数据区域
  • 尾(管理扇区所需要的数据结构)

Step1：低级格式化(物理)，将磁盘的各个磁道划分为扇区
Step2:将磁盘分区，每个分区由若干柱面组成(C，D，E盘)

初始化程序/自举程序：完成计算机开机需要的一系列初始化工作
引导块：位于磁盘的固定位置，存放完整的自举程序

坏块：无法正常使用的扇区
简单磁盘：在FAT上标名(对OS不透明)
复杂磁盘：扇区备用


2.3 减少磁盘延迟时间的方法

一次磁盘读写需要的时间=寻找时间+延迟时间+传输时间

磁头读入一个扇区数据后需要一小段时间处理，逻辑上相邻物理上也相邻，连续读入几个逻辑扇区可能需要很长的延迟时间

交替编号

让逻辑上相邻的扇区物理上有一定间隔，降低延迟

磁盘的物理地址=(柱面号，盘面号，扇区号)

错位命名
