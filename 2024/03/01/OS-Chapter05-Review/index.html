1.1 I/O设备

I/O：输出/输出
I/O设备：将数据输入到计算机/接受计算机输出数据

I/O设备分类

按使用特性分类：

  • 人机交互外部设备(慢)：鼠标，键盘，打印机
  • 存储设备(快)：移动硬盘，光盘
  • 网络通信设备(中)：猫

按传输速率分类：

  • 低速设备
  • 中速设备
  • 高速设备

按信息交换单位分类：

  • 块设备(快)：可寻址
  • 字符设备(慢)：不可寻址

1.2 I/O控制器

I/O设备组成：

  • 机械设备：执行具体的I/O操作，按钮，屏幕
  • 电子部件(I/O控制器或设备控制器)：插入主板扩容槽的印刷电路板，CPU对设备控制的中介

I/O控制器功能：

  • 接受和识别CPU发出的命令：控制寄存器
  • 向CPU报告设备状态：状态寄存器
  • 数据交换：数据寄存器
  • 地址识别

I/O控制器组成

注：

  • 一个I/O设备可能对应多个设备
  • 控制寄存器，状态寄存器，数据寄存器可能多个
  • 寄存器也有相应的地址，寄存器占用内存地址一部分称为内存映像I/O,I/O专用地址称为寄存器独立编址

寄存器编址方式

分类：

  • 内存映像I/O
  • 寄存器独立编址


1.3 I/O设备的控制方式

上一节我们讲了什么东西来控制I/O设备，这节我们将来讲如何控制I/O设备的。

分类：

  • 程序直接控制方式
  • 中断驱动方式
  • DMA方式
  • 通道控制方式

程序直接控制方式

感觉图说的更简单，所以这里就直接贴图了

注：

  • 轮询
  • CPU干预的频率：等待I/O过程中需要不断轮询检查
  • 数据传送的单位：每次读写一个字
  • 数据流向：
    • 读：I/O设备 -> CPU -> 内存
    • 写：内存 -> CPU -> I/O设备
  • 优点：实现简单
  • 缺点：CPU和IO只能串行工作，CPU要一直轮询，长期处于忙等

中断驱动方式

由于程序直接控制方式CPU利用率低、忙等,所以提出了中断驱动方式

思想：引入中断机制，CPU等待IO阻塞切换到别的进程执行，IO完成会向CPU发出中断信号，CPU执行中断处理程序

注：

  • CPU会在每个指令周期某位检查中断
  • 中断处理过程需要保存/恢复进程的运行环境
  • CPU干预的频率：IO开始前、完成后需要CPU介入
  • 数据传送的单位：每次读写一个字
  • 数据流向：
    • 读：I/O设备 -> CPU -> 内存
    • 写：内存 -> CPU -> I/O设备
  • 优点：不需要轮询，CPU和IO可以并行工作
  • 缺点：每次读写一个字都要经过CPU，频繁中断会消耗很多CPU时间

DMA方式

直接存储器存取(DMA):数据传送单位是块,CPU不做中介

DR(数据寄存器) ：暂存从设备<->内存的数据
MAR(内地址寄存器) ：IO时，MAR表示数据应放到内存的什么位置
DC(数据计数器) ：表示剩余要读写的字节数
CR(状态寄存器)：用于存放CPU发来的IO命令

注：

  • CPU干预的频率：仅在传送一个或多个数据块开始前、完成后需要CPU介入
  • 数据传送的单位：每次读写一个或多个块
  • 数据流向：
    • 读：I/O设备 -> 内存
    • 写：内存 -> I/O设备
  • 优点：以块为单位
  • 缺点：CPU每发出一条IO指令，只能读写一个或多个连续的数据块

通道控制方式

由于DMA的缺点只能连续存储，所以通道控制方式应运而生

通道(硬件，弱CPU)：可以识别并执行一系列通道指令

  • CPU干预的频率：通道会根据CPU知识执行相应的通道程序，在传送一个或多个数据块开始前、完成后需要CPU介入
  • 数据传送的单位：每次读写一个或多个块
  • 数据流向(通道控制下)：
    • 读：I/O设备 -> 内存
    • 写：内存 -> I/O设备
  • 优点：CPU、通道、IO可以并行工作
  • 缺点：实现复杂，需要专门的通道硬件支持


1.4 I/O软件的层次结构

用户层软件

用户层软件：实现了与用户交互的接口，例prtinf(“xxx”);将用户请求翻译成格式化的IO请求，并通过system call来请求操作系统内核的服务

设备独立性软件

设备独立性软件/设备无关软件： 完成设备的硬件特性无关的功能实现

主要实现功能：

  • 向上一层提供调用接口system call
  • 设备的保护
  • 差错处理
  • 设备的分配与回收
  • 数据缓冲区管理
  • 建立逻辑设备名到物理设备名的映射；根据设备类型选择调用相应的驱动程序(通过逻辑设备表LUT)

根据设备类型选择调用相应的驱动程序的原因：不同厂家规定寄存器状态不同,硬件特性也不同

管理LUT的方式分类：

  • 整个系统设置一张LUT(单用户操作系统)
  • 每个用户设置一张LUT

设备驱动程序

负责对硬件设备的具体控制，将read/write等转化为听得懂的操作

中断处理程序

2.2 假脱机技术(SPOOLing)

脱机技术

脱机技术：由于手工操作浪费时间，使用磁带完成

假脱机技术

假脱机技术:使用软件的方式来模拟脱机技术

应用：共享打印机

独占和共享设备：只允许个个进程串行使用的设备
共享设备：允许多个进程**同时(宏观)**使用设备


2.3 I/O设备的分配与回收

设备分配时应该考虑的因素：

  • 固有属性
  • 分配算法
  • 分配算法中的安全性

固有属性分类：

  • 独占设备：打印机
  • 共享设备：同是分配给多个进程使用(磁盘)
  • 虚拟设备：SPOOLing

设备分配算法：

  • 先来先服务
  • 优先级
  • 短任务优先

分配算法中的安全性：

  • 安全分配方式：为进程分配一个设备后就将进程阻塞，本次IO完成后才唤醒
    • 优点：不会死锁
    • 缺点：CPU和IO智能串行工作
  • 不安全分配方式：进程发出IO请求，系统分配IO设备，进程还可继续执行
    • 优点：进程的计算任务和IO可以并行
    • 缺点：死锁

静态分配与动态分配

静态分配：进程运行前为其分配全部所需资源，运行后归还资源(不会死锁)
动态分配：进程运行过程中动态申请设备资源

设备分配管理中的数据结构

设备、控制器、通道之间的关系：

设备控制表DCT：系统为每个设备配置一张DCT，用于记录设备情况

控制器控制表COCT：每个设备控制器

通道控制表CHCT：每个通道

系统设备表SDT：系统中全部设备情况

设备分配步骤

  • 根据进程请求的物理设备名查找SDT
  • 根据SDT找到DCT，若设备忙组则将进程的PCB挂到设备等待队列中，不忙碌则将设备分配给进程
  • 根据DCT找到COCT，忙碌则挂到控制器等待队列，不忙组则将控制器分配给进程
  • 根据COCT找到CHCT，忙碌则挂到通道等待队列，不忙组则将通道分配给进程

注：只有设备、控制器、通道三者都分配成功才算成功，之后才启动IO

设备分配的改进步骤

上面方法存在的缺点：

  • 用户编程必须使用物理设备名
  • 换了物理设备程序无法运行
  • 进程请求的物理设备忙碌，同类型的设备也无法使用

改进思想：建立逻辑设备名于物理设备名的映射(逻辑设备表LUT)，用户编程只需要提供逻辑设备名

改进后的步骤：

  • 逻辑设备名找SDT
  • SDT中找设备
  • DCT到COCT
  • COCT到CHCT

2.4 缓冲区管理

缓冲区：存储区域(内存/硬件寄存器)，速度快

缓冲区的作用：

  • 缓和CPU于IO之前速度不匹配
  • 减少对CPU的中断频率
  • 解决数据粒度不匹配
  • 提高CPU于IO的并行性

单缓冲

单缓冲：主存中分配一个缓冲区(默认一个缓冲区=块)
处理数据平均用时:

  • T>C:平均用时=T+M
  • T< C: 平均用时=C+M

注：缓冲区非空不可冲入数据，缓冲区为空充满才可以传出数据

双缓冲

双缓冲：主存中分配两个缓冲区
处理数据平均用时:Max(T,C+M)

单缓冲和双缓冲的区别：

两台机器之间，配置单缓冲区在，任一时刻只能实现数据的单向传输(类似管道)

循环缓冲区

循环缓冲区：多个大小相等的缓冲区连接成一个循环队列

缓冲池

缓冲池：由系统中共用的缓冲池组成

缓冲池按照使用状况分类：

  • 空缓冲队列
  • 输入队列
  • 输出队列

按照功能分类：

  • 收容输入
  • 提取输入
  • 收容输出
  • 提起输出
