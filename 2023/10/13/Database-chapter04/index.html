4.1.3 Outer Joins

The outer-join operation: 通过在包含空值的结果中创建元组来保留在连接中丢失的元组

outer join：

  • left outer join: 仅保留左外连接操作之前（左侧）命名的关系中的元组
  • right outer join: 仅在以右外连接操作命名（右侧）的关系中保留元组
  • full outer join: 完整外连接保留两个关系中的元组

The inner-join operation: 不保留不匹配元组的连接操作

4.2.3 Materialized Views

materialized views: 某些数据库系统允许存储视图关系，但它们确保如果视图定义中使用的实际关系发生变化，视图也保持最新。

view maintenance: 保持物化视图最新的过程称为物化视图维护。当定义视图的任何关系被更新时，可以立即完成视图维护。


4.2.4 Update of a View

尽管视图是一个有用的查询工具，但如果我们用它们表达更新、插入或删除，它们就会带来严重的问题。困难在于，用视图表达的对数据库的修改必须转化为对数据库逻辑模型中实际关系的修改。

定义视图的查询满足以下条件，则称 SQL 视图是updatable:

  • from 子句只有一个数据库关系
  • select 子句仅包含关系的属性名称，没有任何表达式、聚合或不同的规范
  • 任何未在 select 子句中列出的属性都可以设置为 null；也就是说，它没有非空约束，也不是主键的一部分。
  • 该查询没有 group by 或having 子句。

4.3 Transactions

transaction由一系列query和/或update语句组成。SQL标准指定事务在执行SQL语句时隐式开始。SQL语句必须以下列之一结束任务:

  • Commit work: 提交当前事务;也就是说，它使事务执行的更新在数据库中成为永久的。事务提交后，自动启动一个新的事务。
  • Rollback work: 导致回滚当前事务;也就是说，它撤销事务中SQL语句执行的所有更新。因此，数据库状态将恢复到执行事务的第一条语句之前的状态。

关键字work在这两个语句中都是可选的


ACID属性：

  • 原子性（Atomicity） : 事务中的所有操作要么全部完成，要么全部不完成。
  • 一致性（Consistency） : 事务必须确保数据库从一个一致状态转变为另一个一致状态。
  • 隔离性（Isolation） : 事务的执行不会被其他事务干扰。即一个事务的中间状态对其他事务是不可见的。
  • 持久性（Durability） : 一旦事务完成，其修改是永久的，即使系统崩溃也不会丢失。

4.4.5 Referential Integrity

Referential Integrity:确保在一个关系中针对给定的属性(外键)出现的值也出现在另一个关系中的属性(主键)

ON DELETE CASCADE: 当在父表中删除一条记录时，与之相关的子表中的所有记录也会被自动删除

ON UPDATE CASCADE: 当在父表中更新一个主键的值时，子表中引用该主键的所有记录也会自动更新以反映这个更改



主键、候选键和外键可以在create table语句中指定:

  • 主键子句列出构成主键的属性。
  • 唯一键子句列出包含候选键的属性。
  • 外键子句列出组成外键的属性，以及外键引用的关系的名称。

如何插入元组而不造成约束违反:

  • 在插入person之前插入person的父亲和母亲
  • 或者最初将父和母设置为null，在插入所有人员后更新(如果父属性和母属性声明为非null，则不可能实现)

4.5.1 Date and Time Types in SQL

SQL标准支持多种与日期和时间相关的数据类型


日期时间数据类型：

  • data(‘2018-04-25’): 包含（四位数）年、月和日的日历日期
  • time(‘09:30:00’): 一天中的时间，以小时、分钟和秒为单位。变体 time(p) 可用于指定秒的小数位数（默认值为 0）。通过用时区指定时间，还可以将时区信息与时间一起存储。
  • timestamp(‘2018-04-25 10:29:01.45’): 日期和时间的组合。变体timestamp(p) 可用于指定秒的小数位数（此处默认为 6）。如果指定了时区，时区信息也会被存储
  • datetime：MySQL提供的数据类型，代表不考虑时区调整的时间
  • interval:如x - y返回两个日期类型之间的天数间隔

提取日期时间字段：

  • 提取年、月、日、小时、分钟或秒：extract (field from d),field为year，month…
  • 时区信息可通过 timezone hour 和 timezone minute 提取

获取当前日期时间的函数：

  • current date：返回当前日期
  • current time： 返回当前时间（带时区）
  • localtime 返回当前本地时间（不带时区）
  • current timestamp: 返回当前的时间戳（包含时区）
  • localtimestamp: 返回当前的本地日期和时间（不含时区）

4.5.8 Schemas, Catalogs, and Environments

三级层次结构：
现代数据库系统提供一个三级层次结构来组织数据对象：

  • 目录（Catalogs）： 也被称为数据库，每个目录可以包含多个架构（Schemas）。目录是最顶层的组织单位。
  • 架构（Schemas）： 存储关系（表）和视图（Views）。架构对象代表数据库对象的所有权上下文，是组织和管理表、视图等SQL对象的集合。
  • SQL对象： 如关系（表）和视图等。这些是构成数据库内容的基本元素。

三部分名称：

  • 为了在数据库中唯一标识一个关系，可以使用包含三部分的名称，格式通常为：
    catalog5.univ_schema.course
  • 这里catalog5是目录名，univ_schema是架构名，course是关系（表）名。
  • 数据库.gsmCDI_schema.tbcell：这里数据库是目录（或数据库）名，gsmCDI_schema是架构名，tbcell是关系名。

数据库连接：

  • 用户在对数据库执行任何操作之前，必须先建立连接。
  • 连接时，用户需要提供用户名和通常的密码来验证身份。

默认目录和架构：每个用户都有一个默认的目录（Catalog）和架构（Schema），这个组合对每个用户是唯一的。


SQL环境：SQL环境包括用户标识符、连接、目录和架构,为用户与数据库的交互定义了一个上下文，包括身份验证和数据访问的初始设置

4.7 Authorization

对数据的权限：

  • 读取（Read）: 允许用户读取数据，但不允许修改数据。
  • 插入（Insert）: 允许用户插入新数据，但不允许修改现有数据。
  • 更新（Update）: 允许用户修改数据，但不允许删除数据。
  • 删除（Delete）: 允许用户删除数据。

修改数据库架构的权限：

  • 资源（Resources）: 允许用户创建新的关系（表）。
  • 变更（Alteration）: 允许用户在关系（表）中添加或删除属性（字段）。
  • 删除（Drop）: 允许用户删除关系（表）。
  • 索引（Index）: 文章未详述，但通常指允许用户创建或删除索引。

4.8 Summary

连接类型：

  • SQL支持多种类型的连接，包括自然连接、内连接和外连接，以及多种连接条件。
  • 自然连接提供了一种简单的方法来编写跨多个关系的查询，它通过匹配名称的属性来等同化这些属性。
  • join-using 结构提供了一种简单的方法来编写查询，它对于一些具有匹配名称的属性需要等值连接，但不一定是所有这样的属性。
  • join-on 结构提供了一种包含连接谓词在 from 子句中的方法。
  • 外连接提供了一种保留那些因连接谓词而不会出现在结果关系中的元组的方法。保留的元组会用 null 值填充，以符合结果模式。

视图关系： 视图可以定义为包含查询结果的关系。视图对隐藏不需要的信息很有用，并且可以将来自多个关系的信息汇集到一个单一的视图中。

事务： 事务是一系列的查询和更新，它们一起执行一个任务。事务可以被提交或回滚；当事务回滚时，事务执行的所有更新的效果都会被撤销。

完整性约束： 完整性约束确保被授权用户所做的数据库更改不会导致数据一致性的丧失。

引用完整性约束： 引用完整性约束确保出现在一个关系的给定属性集中的值也出现在另一个关系的某个属性集中。

域约束： 域约束指定与属性相关联的可能值的集合。这些约束还可能禁止对特定属性使用 null 值。

断言： 断言是声明性表达式，声明我们要求始终为真的谓词。

SQL数据定义语言： SQL数据定义语言支持定义内置域类型，如日期和时间，以及用户定义的域类型。

索引： 索引对于高效处理查询以及高效执行完整性约束非常重要。虽然索引不是SQL标准的一部分，但大多数数据库系统支持创建索引的SQL命令。

SQL授权机制： SQL授权机制允许区分数据库的用户，在数据库中对各种数据值的访问类型上有所不同。
