1.引言

训练深度神经网络时会遭遇许多困难：

  • 网络后面的层学的很好，但前面的层停止学习
  • 网络前面的层学的很好，但后面的层却停止学习
  • 梯度不稳定导致前面的层易爆炸或弥散

那么是哪些因素导致了上面的这些问题呢？


根据我们的常识，人们通常在出错时学习的更快，所以需要对错误有一个良好的定义


例子：训练下面的网络使得输入1输出0



  • Setting 1: 权重和偏置初始化为2.0，学习速率设为0.15，选择均方误差损失函数
  • 在决策错误的时候，人工神经元很难快速学习



  • Setting 2:权重初始化为0.6，偏置初始化为0.9，学习率设为0.15，选择均方误差
  • 在决策错误时，人工神经元学习的更快

网络优化与正则化

如何解决这种速度放缓：

  • 激活函数
  • 优化算法

1.激活函数

1.1 常见的激活函数

(1)Sigmoid: h=11+e−zh = \frac{1}{1+e^{-z}}h=1+e−z1​

  • 适用条件：
    • Sigmoid 函数的输出范围是 0 到 1。由于输出值限定在 0 到 1，因此它对每个神经元的输出进行了归一化；
    • 用于将预测概率作为输出的模型。由于概率的取值范围是 0 到 1，因此 Sigmoid 函数非常合适；
    • 梯度平滑，避免「跳跃」的输出值；
    • 函数是可微的。这意味着可以找到任意两个点的 sigmoid 曲线的斜率；
    • 明确的预测，即非常接近 1 或 0
  • 缺点：
    • 倾向于梯度消失：输入较大或较小时，输出几乎是平滑的并且梯度较小，这不利于权重更新
    • 函数输出不是以 0 为中心的，这会降低权重更新的效率；
    • Sigmoid 函数执行指数运算，计算机运行得较慢


(2)Tanh: h=ez−e−zez+e−zh = \frac{e^z - e^{-z}}{e^z + e^{-z}}h=ez+e−zez−e−z​

  • 优点：以0为中心
  • 缺点：
    • 梯度消失
    • 幂运算计算量大


(3)RELU: h=max(0,z)h= max(0,z)h=max(0,z)

  • 优点：
    • 解决了正区间内的梯度消失问题
    • 计算速度快：ReLU 函数中只存在线性关系，因此它的计算速度比 sigmoid 和 tanh 更快
  • 缺点：
    • Dead ReLU：某些神经元可能永远不会被激活，导致相应的参数永远不能被更新。有两个主要原因可能导致这种情况产生: (1) 非常不幸的参数初始化，这种情况比较少见 (2) learning rate太高导致在训练过程中参数更新太大，不幸使网络进入这种状态
    • 输出不是以0为中心的


(4)Leaky RELU:

  • 专门为了解决Dead ReLu
  • 而且具有Relu的所有优点，但实际中并未证明总是比ReLU好

1.2 总结

使用ReLU要注意设置lr，不要让训练过程中出现很多dead神经元

dead无法解决，可以尝试Leaky ReLU，PReLU等来替代ReLU

不建议使用sigmoid，如果一定要使用也可以用tanh来替代
