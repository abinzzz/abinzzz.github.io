1.引言

训练深度神经网络时会遭遇许多困难：

  • 网络后面的层学的很好，但前面的层停止学习
  • 网络前面的层学的很好，但后面的层却停止学习
  • 梯度不稳定导致前面的层易爆炸或弥散

那么是哪些因素导致了上面的这些问题呢？


根据我们的常识，人们通常在出错时学习的更快，所以需要对错误有一个良好的定义

1.激活函数

1.1 常见的激活函数

(1)Sigmoid: h=11+e−zh = \frac{1}{1+e^{-z}}h=1+e−z1​

  • 适用条件：
    • Sigmoid 函数的输出范围是 0 到 1。由于输出值限定在 0 到 1，因此它对每个神经元的输出进行了归一化；
    • 用于将预测概率作为输出的模型。由于概率的取值范围是 0 到 1，因此 Sigmoid 函数非常合适；
    • 梯度平滑，避免「跳跃」的输出值；
    • 函数是可微的。这意味着可以找到任意两个点的 sigmoid 曲线的斜率；
    • 明确的预测，即非常接近 1 或 0
  • 缺点：
    • 倾向于梯度消失：输入较大或较小时，输出几乎是平滑的并且梯度较小，这不利于权重更新
    • 函数输出不是以 0 为中心的，这会降低权重更新的效率；
    • Sigmoid 函数执行指数运算，计算机运行得较慢


(2)Tanh: h=ez−e−zez+e−zh = \frac{e^z - e^{-z}}{e^z + e^{-z}}h=ez+e−zez−e−z​

  • 优点：以0为中心
  • 缺点：
    • 梯度消失
    • 幂运算计算量大


(3)RELU: h=max(0,z)h= max(0,z)h=max(0,z)

  • 优点：
    • 解决了正区间内的梯度消失问题
    • 计算速度快：ReLU 函数中只存在线性关系，因此它的计算速度比 sigmoid 和 tanh 更快
  • 缺点：
    • Dead ReLU：某些神经元可能永远不会被激活，导致相应的参数永远不能被更新。有两个主要原因可能导致这种情况产生: (1) 非常不幸的参数初始化，这种情况比较少见 (2) learning rate太高导致在训练过程中参数更新太大，不幸使网络进入这种状态
    • 输出不是以0为中心的


(4)Leaky RELU:

  • 专门为了解决Dead ReLu
  • 而且具有Relu的所有优点，但实际中并未证明总是比ReLU好

h={z,if z≥ 00.01z,if z< 0 h = \begin{cases} z, & \text{if $z \geq$ 0} \\ 0.01z, & \text{if $z <$ 0} \end{cases} h={z,0.01z,​if z≥ 0if z< 0​


1.2 总结

使用ReLU要注意设置lr，不要让训练过程中出现很多dead神经元

dead无法解决，可以尝试Leaky ReLU，PReLU等来替代ReLU

不建议使用sigmoid，如果一定要使用也可以用tanh来替代

2.2 批的大小

批量大小不影响随机梯度的期望，但是会影响随机梯度的方差：

  • 批量越大，随机梯度的方差越小，引入的噪声越小，训练也越稳定，因此可以设置较大的学习率
  • 批量较小，需要设置较小的学习率，否则模型难以收敛

2.3 梯度下降存在的问题

损失函数特性：在一个方向上变化迅速而在另一个方向上变化缓慢

优化目标：从起点处走到终点处

问题：山臂间震荡，往谷底的方向行进缓慢


2.4 动量梯度下降(Momentum)

目标：减小震荡，加速通往谷底

方案：利用累加历史梯度信息更新梯度

现象：损失函数常具有不太好的局部最小值或者鞍点

梯度下降算法的问题：局部最小点和鞍点出梯度都为0

动量法优势：由于动量存在，算法可以冲出局部最小点以及鞍点，找到最优解

2.5 自适应梯度法(AdaGrad)

AdaGrad优势：自适应梯度法通过减小震动方向步长，增大平坦方向步长来减小震荡，加速通往谷底

震荡方向和平坦方向的区分：

  • 梯度幅度的平方较大的方向是震荡方向
  • 梯度振幅的平方较小的方向是平坦方向

方法：

  • 计算每个参数梯度平方的累积
  • 每次迭代时自适应的衰减每个参数的学习率

对数似然函数为:

l(w,b∣X,y)=∑[ylogp+(1−y)log(1−p)] l(w,b|X,y) = ∑[ y logp + (1-y)log(1-p) ] l(w,b∣X,y)=∑[ylogp+(1−y)log(1−p)]


而交叉熵损失定义为:

H(y,p)=−∑[ylogp+(1−y)log(1−p)] H(y,p) = -∑[y logp + (1-y)log(1-p)] H(y,p)=−∑[ylogp+(1−y)log(1−p)]

可见,极大化对数似然l(w,b|X,y)等价于最小化交叉熵损失H(y,p)。


7.3 为什么L1正则化倾向于得到稀疏解？为什么L2正则化倾向于得到平滑解？

L1正则化为什么倾向于得到稀疏解

L1正则化在目标函数中加入了参数的绝对值之和作为正则项,具体形式如:

J(θ)=L(θ)+λ∑i∣θi∣J(\theta) = L(\theta) + \lambda \sum_i |\theta_i| J(θ)=L(θ)+λi∑​∣θi​∣

其中,J(θ)J(\theta)J(θ) 是带正则化的目标函数,L(θ)L(\theta)L(θ) 是原始的损失函数(如均方误差),λ\lambdaλ 是正则化参数,θi\theta_iθi​ 是模型的参数。

由于L1正则化的正则项是参数的绝对值之和,这会导致在优化过程中,一些参数会被压缩到零。这是因为在绝对值函数周围,当参数接近0时,对应的梯度(或子梯度)有一个不连续的变化,使得某些参数在优化过程中直接变为0。这样,L1正则化倾向于产生稀疏解。


L2正则化为什么倾向于得到平滑解

L2正则化在目标函数中加入了参数的平方和作为正则项,具体形式如:

J(θ)=L(θ)+λ∑iθi2J(\theta) = L(\theta) + \lambda \sum_i \theta_i^2 J(θ)=L(θ)+λi∑​θi2​

其中,和上面的符号含义相同。

L2正则化的正则项是参数的平方和,它对所有的参数都施加了“压缩”作用,使参数向0收缩,但并不完全为0。这是因为在平方函数附近,梯度是连续且平滑的,使得参数在优化过程中逐渐收缩但不容易归零。所以,L2正则化倾向于得到平滑的解,所有参数都被“缩小”但大多数都不为零。

从几何的角度看,L1正则化的约束在参数空间中形成了一个“菱形”区域,而L2正则化的约束形成了一个“圆形”或“球形”区域。由于L1的“尖点”在坐标轴上,所以优化的解更容易在这些尖点上,导致某些参数为0;而L2的球形约束使得解更容易分布在球的表面,导致所有参数都有值但都被缩小。


7.4 Batch normalization对参数优化器到什么作用

改进优化器的收敛性：Batch Normalization 通过对每个小批量数据的输入进行标准化，使其均值为0，方差为1，这样使得每层的输入分布更加稳定。这种处理有助于解决所谓的内部协变量移位问题，即网络每层的输入分布在训练过程中不断变化。这种变化会影响每层的学习速度，并可能导致训练变得不稳定。通过BN，模型更容易和更快地收敛。

允许使用较大的学习率：因为BN有助于减少每层的输入分布的变化，它还允许使用更大的学习率而不导致训练发散。

减少对参数初始化的依赖：在没有BN的神经网络中，参数的初始值很关键，不当的初始值可能导致训练过程中的梯度消失或爆炸。而BN，由于其对每一层输入的标准化作用，使得模型对参数初始化方式不那么敏感。

正则化效应：BN的一个有趣的副作用是，它为模型引入了一种轻微的正则化效应。由于在每次训练迭代中使用的是小批量的统计数据来进行标准化，所以这增加了一些噪声。这种噪声有助于防止模型过拟合。

与各种优化器的兼容性：虽然BN与许多优化器都是兼容的，但它与像Adam这样的自适应学习率优化器尤其有效。因为BN可以平滑模型的优化面，使得自适应学习率优化器能够更有效地进行参数更新。
