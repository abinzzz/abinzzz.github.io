2.brew

🔗：Homebrew中文官网

brew是MacOS系统下的一个包管理器,全称是Homebrew。它可以用来在Mac上安装和管理各种开源软件。

特点

  • 简单方便:通过简单的brew install命令就可以安装大量开源软件,不需要了解编译细节。
  • 包依赖管理:Homebrew会自动处理包之间的依赖,递归安装需要的库文件。
  • 二进制预编译:Homebrew会提供预编译二进制文件,无需本地编译源码。
  • 简洁的包:Homebrew安装软件都在独立目录/usr/local/Cellar下,不会污染系统目录。
  • 方便卸载:可以通过brew uninstall轻松卸载不需要的程序。
  • 用户制作的Recipe:任何人都可以为Homebrew制作安装Recipe,方便管理第三方软件。
  • 基于Git:Homebrew使用Git进行软件版本控制和安装源的维护。
  • 免费开源:Homebrew是自由开源软件,由社区维护。

常用命令

  • brew update ：检查和升级Brew。
  • brew outdated ： 检查所安装的软件是否有过期需要升级的。
  • brew upgrade：升级所有已安装的软件，如果只升级某一软件，后面跟软件名即可。
  • brew uninstall 软件名 –force：卸载某一软件。
  • brew cleanup：清理所有已安装的软件的旧版本。
    针对cleanup命令需要说明的是，Brew默认不会卸载软件的旧版本。
  • brew pin 软件名：锁定某一软件为特定版本。(解除锁定用unpin)
    针对pin命令需要说明的是，如果某个软件需要依赖某个被锁定的软件，则被锁定的软件会被强制升级为最新版本。
  • brew –cache：查看所安软件的安装包下载的目录，可以定期清理该目录。
  • brew info 软件名：查询某一所安装的软件信息。

3.npm

npm(Node.js Package Manager)是一个Node.js的包管理工具，用来解决Node.js代码部署问题。在安装Node.js时会自动安装相应的npm版本,不需要单独安装。

使用npm包管理工具可以解决如下场景的需求：
(1）从npm服务器下载别人编写的第三方包到本地使用。
(2）从npm服务器下载并安装别人编写的命令程序到本地使用。
(3）将自己编写的包或命令行程序上传到npm服务器供别人使用

常用指令

npm提供了快速操作包的命令，只需要简单命令就可以很方便地对第三方包进行管理，下面列举了npm中的常用命令。

  • npm install:安装项目所需要的全部包，需要配置 package.json文件。
  • npm uninstall:卸载指定名称的包。
  • npm install包名:安装指定名称的包，后面可以跟参数“-g”表示全局安装“—save表示本地安装。
  • npm update:更新指定名称的包。
  • npm start:项目启动;通过CDN方式引入 Vue，可以缓解服务器的压力，加快文件的下载速度。目前，网络上有很多免费的CDN服务器可以使用。
  • npm run build:项目构建。

4.styler对象

在 Pandas 中,Styler 对象(Styler)是用于自定义设置 DataFrame 显示样式的一个类

主要作用

  • 设置 DataFrame 在输出时的显示格式,例如字体、颜色、背景等。
  • 增加 DataFrame 在 Jupyter Notebook 上的交互功能,例如鼠标悬停高亮。
  • 方便地将 DataFrame 渲染为 HTML 或 Excel 文件。
