1. conda and pip

最佳方法：conda创建虚拟环境，然后使用pip install去下载安装包

conda和pip都是Python的包管理工具,但有以下几点主要区别:

  • 安装源不同：conda默认从Anaconda仓库下载安装包,pip默认从Python官方的PyPI仓库下载。Anaconda仓库包含了更多针对科学计算、数据分析的Python包,PyPI仓库包含了更全面的Python包。

  • 依赖管理：conda可以很好地管理不同Python包之间的依赖关系,避免包冲突。pip的依赖管理不如conda全面。

  • 环境管理：conda可以创建不同的Python虚拟环境,各个环境可以使用不同版本的Python和不同的包。pip可以通过virtualenv等工具创建虚拟环境,但不如conda原生地管理环境。

  • 系统环境：conda可以在Windows、MacOS和Linux上使用,pip也可以跨平台使用但有些区别。

  • 非Python包：conda可以安装并管理一些非Python语言的包,如R语言。pip只能管理Python包。

  • 软件源：conda的仓库源在国内访问可能比较慢,pip的PyPI源存放在国内CDN上速度会快一些。

  • 其他方面：conda对于非conda环境的Python包管理不太方便,pip对环境之间的相互操作更灵活。conda的命令更简单,updgrade可以全部更新。

经验帖

说一点经验吧。大部分情况下，二者没有区别，怎么安装都可以work，不过，二者混着装各种package你真的会因为版本冲突而发疯！conda有严格的检查机制，它会保证你当前装的package安装好之后能work，但是，它只检查用conda安装过的package。例如，你新安装的package会依赖numpy ，不过你已经安装numpy(e.g., 1.19.2)，但是用pip安装的，不好意思，它会认为你没安装，然后用conda再安装一个依赖版本的numpy(e.g., 1.18.5)。这个时候，两个numpy可能就打架了。并且你pip list与conda list显示的numpy版本可能不一致，结果不知道最后运行程序的时候调用了哪个版本的numpy。这个时候，uninstall可能也卸载不干净，卸了一个还有一个，最后还可能一直list有却是一个空壳。这个时候，快刀斩乱麻的方法，找到anaconda的lib/site-packages/numpy, 手动删掉它！pip的一个好处是可以安装时既检查conda安装过package的也检查pip安装过的package。不过，它只负责要什么装什么，不负责能不能把装的一堆packages打通，可能装好不work :(。一般原则，在新环境中，如果装多个packages，既用到conda，又用到pip，那就先conda 的都装好，再pip，如果能用一种装到底，就不要来回换着用。 血泪教训，end.

2. NaN

NaN(Not a Number)表示一个非数值,是Python中的一个特殊浮点数值。

特点

  • NaN与任何值都不相等,包括NaN本身。使用NaN==NaN会返回False。

  • NaN在任何操作中都会返回NaN,例如0 * NaN = NaN。但是NaN与字符串连接时会返回字符串”nan”。

  • 可以用isNaN()或math.isnan()判断一个值是否是NaN。

来源

  • 数值运算中0/0, ∞ - ∞ 等无意义的操作会返回NaN

  • 对非数字类型调用float()会返回NaN

  • 数值运算中溢出或无法表示的结果会返回NaN

3. NaN和NoneType的区别

NoneType和NaN是两个不同的概念:

NoneType

NoneType是Python中None对象的类型。None是一个Python的内置对象,表示不存在的意思,类似于其他语言中的null或者nil。

例如:

NoneType只有一个值即None。判断一个变量是否是None可以用is None或者== None。

NaN

NaN代表”非数值”(Not a Number),是浮点数计算中无法表示的结果的值,比如0/0。NaN属于float类型。

NaN有以下特点:

  • 任何涉及NaN的操作都会返回NaN
  • NaN不等于任何值,包括NaN本身
  • 检测NaN需要使用math.isnan()或isNaN()

区别

None和NaN的主要区别是:

  • NoneType表示不存在的对象,NaN表示无法计算的数值结果
  • None是NoneType类型,NaN是float类型
  • None通常表示空值,NaN表示无效的操作
  • 判断None使用is None, 判断NaN使用isNaN()

缺失值，nan以及nonetype的区别

缺失值是通用概念,NaN和None是具体的实现方法。

6.docker是什么

🔗：[什么是docker？](https://zhuanlan.zhihu.com/p/187505981

Docker是一个开源的应用容器引擎,可以让开发者打包应用以及依赖包到一个可移植的容器中,然后发布到任何流行的 Linux机器或 Windows 机器上。

docker可以屏蔽环境差异，也就是说，只要你的程序打包到了docker中，那么无论运行在什么环境下程序的行为都是一致的，程序员再也无法施展表演才华了，不会再有“在我的环境上可以运行”，真正实现“build once, run everywhere”。

“真正实现“build once, run everywhere””。这句话是错误的，实际上程序员依然有演戏的空间。因为容器只打包了用户空间的系统调用，执行系统调用的地方依然是宿主的kernel，所以当你docker run centos:6 bash执行这句话的时候，在新的内核上可能会发生段错误，而老的宿主机却不会。真正要做到BORE，还是必使用同样的内核，那样的话，和使用虚拟机就没差别了。总之不可能做到开发，测试，运维只要用同一个镜像就能表现完全一致，只要提供系统调用的内核版本不同，嗯，行为就觉得不能保证一致。我就是一个好演员。

Docker 的主要概念包括:

  • 镜像(Image):Docker 镜像是用于创建 Docker 容器的模板,比如一个 Ubuntu 系统。

  • 容器(Container):镜像启动后的实例就称为一个容器,容器是独立运行的一个或一组应用。

  • 仓库(Repository):集中存放镜像文件的场所。Docker Hub 是公共仓库。

Docker 的优点包括:

  • 简化配置:Docker 减少了对环境的要求。

  • 代码连贯性:使得从开发到测试再到部署的代码可以保持一致。

  • 提高开发效率:容器会封装应用的代码和所有依赖项,可以快速直接部署。

  • 隔离应用:容器会将应用进程隔离在容器内,互相不影响。

  • 资源利用:可以提高硬件的利用率,一台主机可以运行多个容器实例。

  • 轻松迁移:可以在不同系统之间轻松迁移,方便水平扩展。

  • 提高安全:容器有自己独立的文件系统,对主机没有访问权,提高安全性。
