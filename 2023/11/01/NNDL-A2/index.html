A2.1 (5 points) 尝试解释Epoch、Iteration、Batch几个概念及其不同，尝试说明batch_size的选择依据和影响。

Epoch：整个数据集通过了神经网络一次正向传递并且返回一次的过程。其数量是指定模型应该在整个训练集上训练多少次。多个Epoch有助于模型学习更多的模式，但是太多的Epoch可能会导致过拟合。

Iteration：单次训练更新的过程。具体来说，每个Iteration是指用一个Batch大小的样本来更新模型的权重。

Batch：一次Iteration中用于更新网络权重的样本集合。

三者之间的关系：如果有1000个训练样本，Batch大小为100，则一个Epoch将包含10个Iterations。

batch_size的选择依据和影响：

  • 硬件资源：Batch Size通常受限于计算机的内存容量。尤其是GPU，它们有限的显存大小决定了能够处理的最大Batch Size
  • 训练的稳定性：较小的Batch Size可能导致训练过程中梯度估计的噪声更大，但也可能有利于模型跳出局部最小值，而较大的Batch Size通常能提供更稳定的梯度下降过程，但计算成本更高。
  • 泛化能力：较小的Batch Size有时能提高模型的泛化能力，因为噪声较多的梯度下降可以视为一种正则化形式。相反，较大的Batch Size可能导致模型过于适应训练数据，而降低泛化性能。
  • 训练速度：较大的Batch Size可以通过并行计算加速训练过程，但这也依赖于硬件的计算能力。

A2.5 (5 points) 分析为什么L1正则化倾向于得到稀疏解、为什么L2正则化倾向于得到平滑的解。

设θ\thetaθ为模型参数,J(θ)J(\theta)J(θ)为原始的损失函数,α\alphaα为正则化系数。

对于L1正则化(Lasso),整体的代价函数为:

L1(θ)=J(θ)+α∑i∣θi∣L_1(\theta) = J(\theta) + \alpha \sum_{i} |\theta_i| L1​(θ)=J(θ)+αi∑​∣θi​∣

对于L2正则化(Ridge),整体的代价函数为:

L2(θ)=J(θ)+α∑iθi2L_2(\theta) = J(\theta) + \alpha \sum_{i} \theta_i^2 L2​(θ)=J(θ)+αi∑​θi2​


在进行梯度下降时,我们需要计算这些代价函数相对于每个参数θi\theta_iθi​的偏导数。

对于L1正则化,偏导数为:

∂L1(θ)∂θi=∂J(θ)∂θi+α⋅sign(θi)\frac{\partial L_1(\theta)}{\partial \theta_i} = \frac{\partial J(\theta)}{\partial \theta_i} + \alpha \cdot \text{sign}(\theta_i) ∂θi​∂L1​(θ)​=∂θi​∂J(θ)​+α⋅sign(θi​)

这里,sign(θi)\text{sign}(\theta_i)sign(θi​)是一个符号函数,当θi>0\theta_i > 0θi​>0时为+1,当θi<0\theta_i < 0θi​<0时为-1,当θi=0\theta_i = 0θi​=0时不可导。由于这个不连续点,梯度下降步骤可能会直接跳过0,导致θi\theta_iθi​变为0,从而产生稀疏解。


对于L2正则化,偏导数为:

∂L2(θ)∂θi=∂J(θ)∂θi+2αθi\frac{\partial L_2(\theta)}{\partial \theta_i} = \frac{\partial J(\theta)}{\partial \theta_i} + 2\alpha\theta_i ∂θi​∂L2​(θ)​=∂θi​∂J(θ)​+2αθi​

在这里,偏导数关于θi\theta_iθi​是平滑的,即使θi\theta_iθi​接近0,这个项仍然存在,会推动θi\theta_iθi​缓慢趋向0而不是直接跳到0。


在权值更新时,对于L1正则化,权值的更新会减去一个与α\alphaα相乘的常数,这容易使得权值达到0;而对于L2正则化,权值的更新会减去与当前θi\theta_iθi​值成比例的量,这使得权值逐渐缩小但不会达到0。这就是为什么L1正则化倾向于得到稀疏解、为什么L2正则化倾向于得到平滑的解。


A2.6 (5 points) 分析Batch normalization对参数优化起到什么作用、如何起到这种作用

BN的作用：

减轻内部协变量偏移：Batch normalization通过规范化每个批次的输入，保持激活分布更加稳定，减轻了这种偏移，允许使用更高的学习率，从而加快了训练速度

允许使用更高的学习率：由于batch normalization有助于保持各层的激活分布比较稳定，因此可以使用更高的学习率而不会那么容易出现梯度消失或梯度爆炸的问题。

降低对初始权重的依赖：由于batch normalization有助于稳定激活的分布，因此网络对权重初始值的敏感度降低。这简化了网络初始化过程并减少了训练失败的风险

提高泛化能力：Batch normalization在每个批次的基础上进行规范化，引入了一些噪声。这种噪声可以看作是一种轻微的数据增强，有助于提高模型的泛化能力。

简化优化过程：Batch normalization改变了loss landscape，使其更加平滑。这有助于梯度下降算法更有效地找到最小值，避免陷入局部极小点。

梯度传播的改善：通过规范化操作，batch normalization有助于减少梯度消失的问题，因为它确保了网络中所有激活都有适当的范围和尺度，从而确保有效的梯度传播。


BN如何起到这种作用

在每次前向传播时，batch normalization通过对小批量的输入进行规范化，即x^(k)=x(k)−μBσB2+ϵ\hat{x}^{(k)} = \frac{x^{(k)} - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}x^(k)=σB2​+ϵ​x(k)−μB​​，然后使用可学习的参数对结果进行缩放和平移，即y(k)=γx^(k)+βy^{(k)} = \gamma \hat{x}^{(k)} + \betay(k)=γx^(k)+β，以此来维持网络的表达能力。

在反向传播时，batch normalization计算出规范化操作对于损失的梯度，并进行相应的参数更新。
