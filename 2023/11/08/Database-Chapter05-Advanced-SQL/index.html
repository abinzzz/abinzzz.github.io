Chapter05: Advanced SQL

本章首先讨论如何使用通用编程语言访问SQL，这对于构建使用数据库管理数据的应用程序非常重要。然后，我们将介绍SQL的一些更高级的特性，从如何通过扩展SQL语言以支持过程操作或允许在数据库中执行过程性代码开始。我们描述了触发器，它可以用来指定在特定事件上自动执行的操作，例如插入、删除或更新指定关系中的元组。最后，我们讨论SQL支持的递归查询和高级聚集特性。


5.1 Accessing SQL from a Programming Language

SQL虽然是一种强大的声明式查询语言，但也需要借助通用编程语言：

  • SQL的局限性：虽然编写SQL查询通常比在通用编程语言中编写相同查询要容易，但SQL并不提供与通用编程语言相同的完全表达能力。
  • 非声明性操作的需求：某些操作，如打印报告、与用户交互或将查询结果发送到图形用户界面，无法仅通过SQL完成。

DBAS = user + application programs + DBMS +DB

API(应用程序接口)与数据库服务器交互

应用程序进行数据处理，并调用:

  • 连接数据库服务器
  • 将SQL命令发送到数据库服务器
  • 将结果元组逐个取出到程序变量中


从通用编程语言中访问SQL有两种方法：

  • 1.dynamic SQL:
    • 通用程序使用函数连接数据库服务器并与之通信
    • 程序将SQL查询构造为字符串，提交查询，然后每次以元组的形式将结果检索到程序变量中
    • 例如：JDBC (Java DB Connectivity) with Java, ODBC (Open DB Connectivity) with C,C++, and Visual Basic
  • 2.embedded SQL:
    • 在高级程序中，SQL语句在编译时被转换为函数调用
    • 这些函数调用使用提供动态SQL功能的API连接到数据库

5.1.1 JDBC(Java Database Connectivity)

JDBC标准定义了一个应用程序接口(API)， Java程序可以使用它连接到数据库服务器。

功能：

  • 支持查询和更新数据。
  • 支持检索查询结果。

元数据检索：

  • 支持获取数据库关系（如表格）信息。
  • 支持查询关系属性的名称和类型。

数据库通信模型：

  • 打开与数据库的连接。
  • 创建SQL语句对象。
  • 使用Statement对象执行查询，发送查询并获取结果。

异常处理：

  • 提供异常处理机制来处理执行SQL时的错误。

使用场景：

  • 连接到数据库，执行CRUD操作。
  • 动态获取数据库结构信息。
  • 应用程序中的错误管理和资源清理。

5.1.1.1 Connecting to the Database

从Java程序访问数据库的第一步是打开到数据库的连接

5.1.1.2 Shipping SQL Statements to the Database System

打开数据库连接后，程序可以使用它将SQL语句发送到数据库系统执行。

发送SQL语句：

  • 通过打开的数据库连接，程序可以发送SQL语句给数据库系统执行。
  • 使用Statement类的实例来发送SQL语句。

getConnection()方法：

  • 存在多个版本，参数不同。
  • 最常用的版本在示例中展示。

驱动定位：

  • 在JDBC 4版本之前，需要手动定位驱动，使用Class.forName。

Statement对象：

  • 是执行SQL语句的工具，并非SQL语句本身。
  • 示例中通过连接对象conn创建了Statement对象stmt。

执行语句：

  • 根据SQL语句的类型（查询或非查询），调用executeQuery()或executeUpdate()方法。
  • stmt.executeUpdate()用于执行更新操作，如插入、更新或删除。
  • 返回影响的行数，对DDL语句返回0。

5.1.1.3 Exceptions and Resource Management

异常处理：

  • SQL方法执行可能抛出异常。
  • 使用try-catch结构来捕获和处理JDBC调用中的异常。
  • 区分SQLException（特定于SQL的异常）与Exception（通用Java异常，如空指针异常）。

资源管理：

  • JDBC对象（连接、声明等）消耗系统资源。
  • 程序应确保关闭所有资源，防止资源泄漏导致数据库系统资源耗尽。
  • 资源未关闭可能导致系统不可访问或操作。

资源关闭机制：

  • 传统方法是显式调用关闭方法，如conn.close()。
  • try-with-resources结构是首选，因为它在try块结束时自动关闭资源。
  • 此结构避免因异常退出而留下未关闭的资源。

try-with-resources优势：

  • 自动管理资源关闭，减少错误和遗漏。
  • 使代码更简洁，避免冗余的关闭语句。

5.1.1.4 Retrieving the Result of a Query

执行查询：使用Statement对象的executeQuery()方法执行SQL查询。

获取结果集：查询结果被检索到一个ResultSet对象rset中，这个对象包含了结果集中的多个元组（行）。

逐行获取数据：通过ResultSet对象的next()方法来逐行遍历结果集。next()方法检查结果集中是否还有未获取的元组，如果有，它会获取下一个元组。next()方法返回一个布尔值，指示是否成功获取到了元组。

提取元组属性：一旦获取到元组，可以使用以get开头的各种方法来提取该元组的属性。例如，getString()方法可以检索基本的SQL数据类型并将其转换为Java字符串对象，而getFloat()等更具体的方法用于检索浮点类型的数据。


5.1.1.5 Prepared Statements

预备语句的创建：预备语句允许在SQL查询中使用问号（?）作为占位符，这些占位符将在后续被实际的参数值替换。当预备语句被创建时，数据库系统会编译这个查询。

执行预备语句：每次执行预备语句时，数据库系统可以重用之前编译的查询形式，并将新的参数值应用到查询中。通过PreparedStatement对象的executeQuery()和executeUpdate()方法来执行查询。

设置参数值：在执行查询之前，必须为占位符设置具体的参数值。使用setString()、setInt()等方法为每个问号占位符分配适当的值。方法的第一个参数是占位符的位置（从1开始计数），第二个参数是要分配的值。


-------------------- Prepared Statements·expmale·BEGIN --------------------

这段话描述的是在Java代码中使用JDBC API创建并执行预备语句（PreparedStatement）的过程，主要用于向数据库中插入数据。

创建预备语句：conn.prepareStatement("insert into instructor values(?,?,?,?)");

  • 这一行代码通过数据库连接对象conn创建了一个预备语句。"insert into instructor values(?,?,?,?)"是一个SQL插入命令，用于向instructor表中插入新数据。问号（?）是参数占位符，它们将被后续指定的参数值替换。

设置参数值：

  • pStmt.setString(1, "88877");
    这一行代码设置了第一个占位符的值为字符串"88877"。
  • pStmt.setString(2, "Perry");
    这一行代码设置了第二个占位符的值为字符串"Perry"。
  • pStmt.setString(3, "Finance");
    这一行代码设置了第三个占位符的值为字符串"Finance"。
  • pStmt.setInt(4, 125000);
    这一行代码设置了第四个占位符的值为整数125000。

执行预备语句：

  • pStmt.executeUpdate();
    这一行代码执行预备语句，将前面设置的参数值插入到instructor表中。此操作不返回任何结果集，但会返回一个整数，表示受影响的行数（通常是插入的行数）。

重新设置参数并再次执行：

  • pStmt.setString(1, "88878");
    这一行代码改变了第一个参数的值，将其设置为"88878"。
  • pStmt.executeUpdate();
    然后再次执行预备语句，这次是将更新后的参数值插入到数据库中。

-------------------- Prepared Statements·expmale·END --------------------


5.1.1.6 Callable Statements

CallableStatement接口:用于从Java程序中调用数据库的存储过程和函数，并处理它们的输入输出参数

CallableStatement接口的作用：CallableStatement允许开发者在Java代码中调用数据库中定义的存储过程和函数。

创建CallableStatement对象：通过Connection对象的prepareCall()方法创建CallableStatement对象。在这个方法中，可以写入调用存储过程或函数的SQL语句，使用问号（?）作为输入参数（IN参数）和输出参数（OUT参数）的占位符。例如：

  • conn.prepareCall("{? = call some_function(?)}"); 这行代码准备调用一个函数，该函数有一个输入参数和一个返回值。
  • conn.prepareCall("{call some_procedure(?,?)}"); 这行代码准备调用一个存储过程，该过程有两个输入参数。

注册输出参数：对于存储过程的输出参数（OUT参数）和函数的返回值，必须使用registerOutParameter()方法来指定其数据类型。这样，当存储过程或函数执行后，可以通过类似于处理结果集的get方法来检索这些值。

检索输出值：执行完存储过程或函数后，可以通过CallableStatement对象的get方法（例如getInt()，getString()等）来获取这些输出参数的值。


5.1.1.7 Metadata Features

Metadata：关于数据的数据。更具体地说，它是描述数据库结构的数据，如表格、列、数据类型和约束的详细信息


用途：

  • 允许Java程序在运行时获取数据库结构信息。
  • 增强应用程序对数据库模式变化的适应性。

结果集元数据（ResultSetMetaData）：

  • 通过ResultSet.getMetaData()获取。
  • 包含列数、列名、列类型等信息。
  • 用于处理未知结构的结果集。

数据库元数据（DatabaseMetaData）：

  • 通过Connection.getMetaData()获取。
  • 提供数据库系统支持的特性、表、列、主键、外键等信息。
  • 用于编写不依赖于硬编码数据库结构的通用代码。

检索列信息：

  • 使用DatabaseMetaData.getColumns()来获取列的详细信息。
  • 参数包括目录名、模式名模式、表名模式、列名模式。
  • 支持SQL通配符，如“%”匹配所有名称。

检索表信息：

  • 使用DatabaseMetaData.getTables()来获取表的列表。
  • 可以限制返回特定类型的表。

-------------------- Prepared Statements·expmale·BEGIN --------------------

获取数据库元数据对象：

  • DatabaseMetaData dbmd = conn.getMetaData();
    这行代码通过已经建立的数据库连接对象conn获取了数据库元数据对象dbmd。

检索表的列信息：

  • ResultSet rs = dbmd.getColumns(null, "univdb", "department", "%");
    这行代码调用getColumns方法来获取名为"department"的表的所有列的信息。方法参数说明如下：
    • 第一个参数是目录名称，这里传入null表示目录名称不作为过滤条件。
    • 第二个参数是数据库模式名称，这里传入"univdb"表示特定的数据库模式。
    • 第三个参数是表名模式，这里传入"department"表示我们正在查询的表名。
    • 第四个参数是列名模式，这里传入"%"表示我们希望获取所有列的信息（%是SQL中的通配符，匹配任意字符）。

遍历结果集并打印信息：

  • while( rs.next()) { ... }
    这个while循环遍历查询返回的每个列的信息。rs.next()方法移动到结果集的下一行，如果有更多行返回true，否则返回false。
  • System.out.println(rs.getString("COLUMN NAME"), rs.getString("TYPE NAME"));
    在循环体内，对于结果集的当前行，使用getString方法分别打印出列名和列的数据类型。"COLUMN NAME"和"TYPE NAME"是getColumns方法返回的结果集的列标签，分别代表列的名称和列的数据类型。

-------------------- Prepared Statements·expmale·END --------------------


5.1.1.8 Other Features

可更新结果集：

  • 支持直接在结果集上修改数据。
  • 修改反映到数据库对应行。

事务控制：

  • setAutoCommit() 方法控制自动提交行为。
  • commit() 和 rollback() 方法显式事务管理。

大型对象处理：

  • getBlob() 和 getClob() 方法获取大型对象。
  • 返回Blob和Clob对象，作为数据库中大型对象的指针。
  • getBytes() 和 getSubString() 方法提取LOB数据。

存储大型对象：

  • setBlob() 和 setClob() 将输入流数据写入BLOB/CLOB列。

RowSet特性：

  • 结果集可以被传递、共享和修改。
  • 支持前向和后向扫描。

5.1.3 ODBC(Open Database Connectivity)

开放数据库连接(ODBC)标准：

  • 用于应用程序(作为客户端)与DB服务器通信。
  • 应用程序编程接口(API)
    • 打开与数据库的连接
    • 发送查询和更新
    • 获取结果
