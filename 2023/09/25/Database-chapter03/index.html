Chapter03: Introduction to SQL

虽然我们将SQL语言称为“查询语言”，但它能做的远不止查询数据库这么简单。它可以定义数据的结构，修改数据库中的数据，并指定安全约束。


3.1 Overview of SQL

SQL:Structured Query language

SQL的组成：

  • Data-manipulation language (DML)：SQL DML提供了从DB中查询信息、向DB中插入tuple、从数据库中删除tuple和修改tuple的能力。
  • Data-deﬁnition language (DDL)：SQL DDL提供了定义relation schemas、deleting relations和修改relation shcemas的命令。
  • Integrity:SQL DDL包括用于指定存储在数据库中的数据必须满足的完整性约束的命令。
  • View deﬁnition:SQL DDL包括用于定义View的命令。
  • Authorization:SQL DDL包括用于指定对关系和视图的访问权限的命令。
  • Transaction control:SQL包含用于指定事务开始和结束的命令
  • Embedded SQL and dynamic SQL:嵌入式和动态SQL定义了如何在通用编程语言(如C、c++和Java)中嵌入sql语句。

3.2 SQL Data Definition

DB中的relation集是使用数据定义语言(DDL)指定的

SQL DDL不仅允许指定一组关系，还允许指定关于每个关系的信息，包括：

  • 定义每个relation的schema
  • 定义与每个attribute关联的值的类型。
  • 添加The integrity constraints,age>0
  • 创建每个relation维护的索引集
  • 每个relation的安全和授权信息。
  • 指定每个relation在磁盘上的物理存储结构

3.2.1 Basic Types

SQL标准支持多种内置类型:

  • char(n):固定长度的字符串，用户指定长度n。可以使用完整形式character代替。
  • varchar(n):具有用户指定的最大长度的可变长度字符串。完整的形式character varying是等效的。
  • int:一个整数(与机器相关的整数的有限子集)。integer形式是等价的
  • smallint:一个小整数(整数类型的与机器相关的子集)
  • numeric(p, d):具有用户指定精度的定点数字。该数由p位(加上一个符号)组成，p位中的d位在小数点的右边。因此，numeric(3,1)允许精确存储44.5，但444.5和0.32都不能精确存储在这种类型的字段中。
  • real, double precision:具有机器相关精度的浮点数和双精度浮点数
  • ﬂoat(n):精度至少为n位的浮点数
  • date:日期，包含 (4位)年 、月和日期
  • time:一天中的时间，以小时、分钟和秒为单位
  • timestamp:日期+时间
  • Interval:一段时间
  • Null:每种类型都可以包含一个特殊的值，称为null值。null值表示一个缺失值，该值可能存在但未知，或者根本不存在。

注意：

  • 在Oracle等大型商用数据库系统中，关系表的属性的名字只能取英文名，不支持中文属性名
  • SQL Server支持中英文属性名
  • 开发大型数据库应用时，程序所访问的关系表的属性名字（最好）取为英文名，便于应用程序的可移植性！
  • 准确表示英文专业术语（英文属性名称), e.g. 分组 packet

3.6 Null Values

空值在关系运算（包括算术运算、比较运算和集合运算）中带来特殊问题。

null:

  • 是未知的值
  • 是不存在的值

null在布尔运算时的情况：

  • and:
    • true and unknown = unknown
    • false and unknown = false
    • unknown and unknown = unknown
  • or:
    • true or unknown = true
    • false or unknown = unknown
    • unknown or unknown = unknown
  • not:
    • not unknown = unknown

🌟 tips:

  • 任何包含null的表达式的结果都是null
  • 将null与其他值进行比较会返回unkown
  • where子句谓词的计算结果为unknown，则将其视为false
  • 对于value的描述用null，对于布尔值用unknown

聚合、group by 或having 子句的查询的含义由以下操作序列定义：

  • from:首先评估 from 子句以获得relation
  • where:where 子句中的谓词将应用于 from 子句的结果关系。
  • group by:满足 where 谓词的tuple将通过 group by 子句（如果存在）放入group中
  • having:having 子句（如果存在）将应用于每个group；不满足having子句谓词的group被删除
  • select:select 子句使用剩余的组生成查询结果的元组，应用聚合函数为每个组获取单个结果元组。

    • 数据定义语言（DDL）:提供定义关系模式、删除关系和修改关系模式的命令
    • 数据操作语言 (DML):包括查询语言和在数据库中插入元组、删除元组和修改元组的命令
  • SQL 数据定义语言用于创建与指定模式的关系。除了指定关系属性的名称和类型之外，SQL 还允许指定完整性约束，例如主键约束和外键约束。
  • SQL 包括多种用于数据库查询的语言结构。其中包括 select、from 和 where 子句。
  • SQL 还提供了重命名属性和关系的机制，以及通过对指定属性进行排序来对查询结果进行排序的机制。
  • SQL支持关系上的基本集合运算，包括并、交和除，它们对应于数学集合运算∪、∩和−。
  • SQL 通过将真值“未知”添加到通常的真值 true 和 false 来处理对包含空值的关系的查询。
  • SQL 支持聚合，包括将关系分组、对每个组单独应用聚合的能力。 SQL 还支持组上的集合操作。
  • SQL 支持外部查询的 where 和 from 子句中的嵌套子查询。只要允许表达式返回值，它还支持标量子查询。
  • SQL 提供了更新、插入和删除信息的结构.
