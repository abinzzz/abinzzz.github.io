Step2: Familiarize yourself with The BLITZ Tools

在本课程中，您将使用BLITZ命令行工具开发您的实验室解决方案，其中

使用称为KPL。在闪电战的任何工具中都不会有图形用户界面。

以下是构成BLITZ工具集的程序:

  • kpl The KPL compiler
  • asm The BLITZ assembler
  • lddd The BLITZ linker
  • blitz The BLITZ machine emulator (the virtual machine and debugger)
  • diskUtil A utility to manipulate the simulated BLITZ “DISK” file
  • dumpObj A utility to print BLITZ .o and a.out files
  • hexdump A utility to print any file in hex
  • check A utility to run through a file looking for problem ASCII characters
  • endian A utility to determine if this machine is Big or Little Endian

这些工具按照使用的大致顺序列出。您可能只需要使用
前4或5个工具，而可以忽略其余工具。(最后三个工具在源代码文件开头的注释中只列出了
，如果您有兴趣，可以阅读这些注释。)

The “makefile”

p1目录包含一个名为makefile的文件，它与UNIX make命令一起使用。当p1目录中的 文件被更改时，您可以键入“make”来重新编译、重新组装和重新链接 ，以重新构建可执行文件。

注意这个命令:

将在文件System.h发生更改时执行。在KPL中，以“。h”结尾的文件称为“头文件”
，以“。c”结尾的文件称为“代码文件”。每个包(例如HelloWorld)都有一个头文件
和一个代码文件。HelloWorld包使用的是System包。只要HelloWorld使用的
包的头文件发生更改，HelloWorld就必须重新编译。但是，如果更改了System的代码文件
，则不需要重新编译HelloWorld。您只需要重新链接(即，您只需要
调用lddd来生成可执行文件)。

有关单独编译包的更多信息，请参阅KPL文档。

The “DISK” File

KPL虚拟机(称为“blitz”的仿真器工具)模拟一个虚拟磁盘。虚拟磁盘
是通过主机上的一个文件实现的，这个文件称为“disk”。项目1中的程序do
不使用磁盘，因此不需要这个文件。但是，如果文件丢失，模拟器将打印
警告。我们已经包含了一个名为“DISK”的文件来防止这种警告。有关更多信息，请尝试在模拟器中使用
" format "命令。
