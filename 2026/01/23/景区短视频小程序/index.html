需求清单

一级模块 二级模块 功能描述 优先级 用户角色 关键约束 / 备注
前端交互层 扫码识别模块 提供标准化扫码入口，支持景区专属二维码识别，触发人脸识别流程；扫码响应时间≤2 秒 P0 游客 需适配不同光线环境下的扫码识别，二维码含景区设备唯一标识
前端交互层 人脸识别模块 展示人脸采集框，实时引导游客完成面部对齐；采集完成后显示 “匹配中” 状态提示，同步上传面部特征值 P0 游客 需弹窗告知《人脸识别授权协议》，获得授权后才可采集；支持戴口罩场景识别优化
前端交互层 主界面载体 基于微信小程序 / H5 构建，包含功能导航栏（首页 / 我的视频 / 帮助中心）、特效模板推荐区、活动公告栏；适配 iOS/Android 主流机型及微信内置浏览器 P0 游客 首屏加载时间≤3 秒，界面符合微信小程序设计规范
前端交互层 用户账号模块 支持微信快捷登录（一键授权）、手机号注册（验证码验证）；登录后显示用户头像、已购视频列表、消费记录 P1 游客 与后端账号系统实时同步用户状态，支持账号注销功能
前端交互层 视频预览模块 提供视频播放器（支持进度拖动、倍速播放），展示合成后的特效视频；播放器下方显示美颜调节入口、特效切换按钮、背景音乐选择栏 P0 游客 视频加载缓冲时间≤1.5 秒，支持 720P 预览画质
前端交互层 支付下载模块 展示阶梯定价面板（720P 水印版免费 / 1080P 无水印 XX 元 / 4KXX 元 / 8KXX 元 / 原始素材 XX 元）；支持微信支付，支付完成后显示下载按钮，支持断点续传 P0 游客 价格需动态配置，支付成功后触发视频权限解锁；下载速度≥5MB/s（5G 环境）
后端服务层 账号管理系统 实现用户信息存储（手机号 / 微信 OpenID / 头像）、子账号授权管理（支持 N 个子账号分配，可设置权限有效期）；提供账号冻结 / 解冻、密码重置接口 P1 管理员 / 游客 子账号权限范围：仅查看权限 / 下载权限 / 管理权限；支持批量授权导出
后端服务层 视频处理系统 接收前端请求后，调用 AI 合成引擎：匹配游客影像片段（从摄像机实时流中提取）→ 与选定特效模板融合→ 叠加背景音乐→ 输出 4K 成片；支持并发处理≥50 路视频合成 P0 系统自动 特效模板库支持动态更新（新增 / 下架）；视频合成耗时≤10 秒
后端服务层 支付系统 对接微信支付 API，实现订单生成（含唯一订单号）、支付回调验证、订单状态同步；支持订单查询、退款申请处理 P0 系统自动 支付数据加密存储，支持对账接口；退款审核流程需人工确认（管理员操作）
后端服务层 数据分析系统 采集用户行为数据（扫码次数 / 模板选择偏好 / 支付转化率）、系统性能数据（合成成功率 / 响应时间）；生成日 / 周 / 月报表，支持数据可视化展示 P2 管理员 数据采样频率≥1 次 / 分钟，支持异常数据告警（如合成失败率＞5%）
后端服务层 数据存储系统 采用分布式存储架构，存储用户视频（24 小时自动清理）、用户基础信息（长期存储）、交易记录（永久存储）；支持视频分片存储和快速检索 P0 系统自动 视频存储加密，清理机制需日志记录；存储空间支持弹性扩容
后端服务层 财务管理系统 自动统计交易流水（收入 / 退款）、生成财务报表（按景区 / 按时间段）；支持财务数据导出（Excel 格式），数据保留周期为 2 年 P1 管理员 报表需包含税费计算、环比增长率分析；支持多景区财务数据隔离
核心功能层 人脸识别与视频捕捉 通过边缘计算节点实时分析摄像机画面，基于人脸识别匹配游客轨迹；自动提取 30 秒最佳片段（含表情 / 动作亮点），与模板合成 60 秒内视频 P0 系统自动 匹配准确率≥95%；支持多摄像机画面联动拼接（如景区多个景点画面融合）
核心功能层 特效合成引擎 内置主题特效模板库（过山车 / 大喇叭 / 花海等），支持 AI 智能匹配场景（如根据游客位置自动推荐对应景点模板）；提供背景音乐库（按风格分类：动感 / 抒情 / 国风），支持音量调节 P0 游客 / 系统自动 模板更新频率≥每月 2 款；支持用户自定义特效参数（如特效强度）
核心功能层 智能美颜系统 基于深度学习算法实现面部特征点检测，提供基础美颜参数（磨皮 / 瘦脸 / 大眼 / 美白），支持强度调节（0-100 档）；预览界面实时显示美颜效果，与最终视频效果一致 P1 游客 美颜处理不影响背景画质；支持多人美颜（如游客与同伴同框时分别处理）
核心功能层 隐私保护模块 自动识别视频中除主角外的其他游客面部 / 身形，采用高斯模糊处理；支持用户手动标记需模糊区域（如背景中的路人） P1 系统自动 / 游客 模糊处理不影响视频主体内容；符合《个人信息保护法》对人脸信息处理的要求
技术支撑层 硬件适配方案 兼容 8K 超高清摄像机（优先 Sony ILME-FR7），支持 RTSP 实时流接入；部署边缘计算服务器（GPU≥NVIDIA T4），保障本地视频处理低延迟；依赖 5G 网络实现设备间数据传输（带宽≥100Mbps） P0 运维人员 服务器需支持 7x24 小时稳定运行；摄像机安装位置需覆盖景区核心游览路线
技术支撑层 核心算法引擎 美颜算法：支持实时面部特征提取，处理延迟≤50ms；视频合成算法：实现模板与实时视频的动态融合（含转场特效 / 字幕叠加），输出视频帧率≥30fps；AI 视角优化：自动调整镜头焦点（突出游客主体） P0 技术开发 算法需定期迭代优化；支持离线备份算法模型（防止网络中断时功能降级）

优先级定义说明

  • P0（最高优先级）：核心功能，直接影响小程序可用性，必须实现且上线前需完全验证
  • P1（高优先级）：重要功能，提升用户体验或运营效率，需在首个正式版本实现
  • P2（中优先级）：优化类功能，可在后续版本迭代中逐步实现
