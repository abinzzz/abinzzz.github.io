1.前言

两种大语言模型：

  • Base LLM
  • Instruction Tuned LLM

Instruction Tuned LLM通常是在大量文本数据上训练过，然后使用输入输出指令来进行微调,使用rhlf进行进一步优化.

一.基础知识

1.计算机时如何运行游戏的

(1)CPU的本质
    CPU ——> 逻辑电路 ——> 逻辑运算 ——> 只能做简单的事情
(2) CPU的工作方式

(3)数据和指令编码
数据的编码：1101= 1x8 + 1x4 + 0x2 + 1x1=13
指令的编码：11 01 00 0110 = add A B 存储到C
如何区分数据和指令：靠上下文信息判断编码含义

二.CheatEngine通关

1.选择进程

通过点击一个类似于电脑显示屏的按钮来选择自己调试的程序(xxx.exe).

2.精确值扫描(已知初始值)

(1)将当前生命值输入到数值中
(2)扫描类型=精确扫描
    数值类型=4字节
(3)点击首次扫描，会发现很多地址
(4)然后改变生命值，将改变后的生命值输入到数值中，点击再次扫描
(5)重复上一步，直到只剩下一个地址，这个地址就是生命值的地址

3.未知的初始值与减少的数值

从上一次扫描中退出要点新的扫描

(1)扫描类型=未知初始值
(2)点击首次扫描，会发现很多地址
(3)然后减少生命值，扫描类型=减少的数值，点击再次扫描
(4)重复上一步，直到只剩下一个地址

4.单精度与双精度

这么做是为了给菜鸟制造一些麻烦，让他们没那么容易修改游戏
(1)选择精确扫描或者未知初始值
(2)数据类型=单浮点/双浮点
(3)然后减少生命值，扫描类型=减少的数值，点击再次扫描
(4)重复上一步，直到只剩下一个地址

5.代码查找

某些游戏开始的时候，数据会存储在与上次不同的地方，甚至游戏过程中数据的位置也会改变
(1)首先要找到这个数值当前的存储地址
(2)右键地址，点击找出是什么改写了这个地址
(3)改变数值，然后窗口内会弹出一些汇编代码
(4)选中代码，点击replace，替换为什么也不做的代码(nop)
(5)这样当程序读到它的时候，将什么也不做

6.指针

(1)找到数值的存储地址，点击找出是什么改写了这个地址
(2)改变数值，就可以看到汇编代码，点击详情信息会告诉你指令运行发生了什么,rdx就是取地址里的内容
(3)使用十六进制去搜索该数值的存储地址，发现有变绿色的，点击手动添加地址,将激活打X就是锁定，该指针锁定的话，数值和指针都不会改变

7.代码注入

将代码注入到目标进程中并执行它的技巧
(1)首先找到数值的存储地址，点击找出是什么改写了这个地址
(2)改变数值，找到改变数值的那条汇编代码，选择反汇编程序
(3)sub是减少的操作码，所以减少生命值的命令就在这里
(4)工具->Auto Assemble->Template，然后添加add的汇编指令即可

8.多级指针

在6中，你已经清楚1级指针的概念和用途，并可以利用数值的首个地址来找到存放数据真正的基址。在本关中，你将看到4级指针，它由一个第一个指针指向第二个指针，再由第二个指针指向第三个指针，由第三个指针指向第四个指针，最终指向健康值的真正地址。

9.注入++之共享代码

解释如何处理游戏中的共用代码, 这种代码是通用在除了自己以外的其他同类型对象上
(1)在改变数值进行精确搜索，但没有结果，可能是因为数值类型并不是默认的”4字节”，而是浮点型

Reading 《Causal Analysis of Customer Churn Using Deep Learning》

一.摘要

  本文提出了一种基于深度前馈神经网络的高维稀疏数据分类框架和序列模式挖掘方法。我们还提出了一个因果贝叶斯网络来预测导致客户流失的原因概率。
  本文提供了一个现实世界的客户流失分析，从现状推断到未来的方向
  关键词:流失分析，因果分析，深度神经分析网络、数据挖掘

二.介绍

探讨了客户流失（churn）分析的重要性，以及如何通过深度学习方法来预测和解决客户流失问题。

  文章提出了一个应用于本地养老基金客户流失分析的框架。通过数据准备和分析，验证了大部分客户账户的活跃期少于一年，原因可能包括一些客户是非自愿流失者，他们随后被注册到由他们的雇主运营的其他养老基金。

  文章的主要贡献是建立了一个特定的倾向性模型框架来分析客户的财务行为数据，然后将模型泛化为任何养老基金业务。首先，分析了12个月的时间窗口数据以预测接下来六个月的客户流失。然后，通过利用贝叶斯网络扩展模型，描述了由于多种原因而发生故意流失的情况，并表明对常见特征的假设检验对预测结果有很大影响。

  最后，文章指出这项研究只关注了DFF神经网络在处理大规模高维稀疏数据集上的表现，这些数据主要在订阅式业务中生成，尤其是在养老基金中，因为它们使用了客户关系管理（CRM）系统中的间隔特性。

三.相关工作

讨论了以往的客户流失（churn）研究，以及作者提出的一个新方法来改进客户流失预测模型。

  大多数以前的研究都专注于确定特定数据集的流失变量，而不是客户流失的因果分析。CHAMP是一个预测消费者取消他们的手机服务的综合系统。Alyuda Neuro-Intelligence使用神经网络进行数据挖掘以预测银行的客户流失。在客户流失预测算法中，结合文本数据可以增加价值，将不同的分类器（例如，梯度提升、过采样和单年观察窗口的对比顺序模式挖掘）结合起来，已经被证明是处理来自养老基金收集的高偏斜数据的实用策略。

  对于养老基金，”隐藏流失”是一个常见问题，即当强制性的雇主支付停止后，客户账户就会变得不活跃。有人提出了各种对不足的消费者互动的补救措施，解决了不平衡和完全利用的数据问题，并从采样数据集开发了多个分类器。深度学习技术可以处理非常大的数据集，相比之下，标准的机器学习方法就显得力不从心。深度学习和卷积神经网络的结合已经成功地预测了客户流失。

  最近的因果推断发展突出了从标准统计分析转向多变量因果分析所需的基本改变。新的PC-Stable方法有效地使用DFS数据学习因果结构，允许对巨大的时间信息数据集进行时间因果建模。已经提出了使用贝叶斯网络表示因果关系的有向无环图。预计会有客户流失的可能性，以及驱动银行数据的因素。

  以上的方法已经被广泛地用于当前的客户流失预测方法，但大多数用于从电信、银行和零售市场收集数据。与此相反，作者提出的方法是使用大规模的高维稀疏数据进行测试。此外，据作者所知，以前没有研究考虑过对养老金公司的客户流失进行因果分析。受到上述研究的启发，作者提出了一种新的方法，该方法结合深度学习和贝叶斯理论来改进客户流失预测建模系统。

四.预备知识

讨论客户流失预测模型的作用和优点，并提出了一种深度学习算法来处理大规模金融数据。

  客户流失预测模型可以提前识别可能的流失客户，从而帮助开发客户干预计划。这本质上是一个分类问题，即将每个客户分类为潜在的流失客户或非流失客户。虽然一些机器学习技术已经产生了可验证的结果，如提升技术、非参数方法、逻辑回归，但这些方法只在客户数据库非常小、样本大小变化的情况下有效，并且在处理更大的数据集时会失败。

  因此，作者提出了一种深度学习算法来处理大量的金融数据，因为深度学习（特征转换）会根据历史数据对特征进行不同的权重分配。深度学习算法使用人工神经网络（ANNs）来模仿人脑的结构。前馈神经网络将一个层的输出作为下一个层的输入，层与层之间没有循环。因此，作者使用深度学习前馈（DFF）算法进行实验，在减少计算负担和不同算法的最高准确度之间进行权衡。DFF神经网络的一般优点包括：

  • 来自深度学习的优越精度；
  • 包含的变量比传统的机器学习更多；
  • 深度学习算法能够从大量的客户人口统计数据、行为变量和数十亿的客户参与日志中提取模式，避免了盲点；
  • 减少了耗时的特征工程和手动金融数据分析。

  作者采用DFF NN方法不仅是因为深度学习的有用特性，而且我们只需要更改最后一层来为未来的数据更新模型，省去了耗时的模型训练。深度学习的一个重要弱点是它仍然是一个黑箱模型，即深度学习无法以易于理解的方式表达数据中的模式。然而，作者进行了实验分析，使用贝叶斯网络来识别客户流失的影响和可能的原因，以解决神经网络模型的复杂性。

五.方法论

1.问题的定义

  文章中使用了一家养老金公司的12个数据集进行实验，以验证所提出模型的有效性。如果客户在接下来的6个月时间窗口内关闭他们的账户，他们就被定义为流失客户。因此，每个客户的结果都是二进制的[0或1]，其中1表示账户已关闭，0表示在接下来的6个月时间窗口内未关闭。

  作者应用了两个主要的包含标准来满足模型开发的需求。首先，只保留了账户存续期超过6个月的客户的数据；其次，为了提高预测准确性，移除了账户余额低于$1500的数据，因为预测不活跃账户的流失概率对养老金基金的价值较低。在观察窗口内的特性可以预测哪个用户在接下来的6个月结果窗口内会流失或者不会流失。

  此外，我们使用了scikit-learn库中的多数投票集成来合并多个模型的预测，这可以帮助提高分类任务的模型性能。在比较监督模型上应用了集成硬投票和软投票。硬投票计算每个单独分类器的投票，多数票获胜。相反，软投票通过分类器的重要性对每个预测进行加权，具有最大加权概率总和的目标标签获胜。

1.数据集

  实验在12个数据集上进行，这些数据集是由当地一家金融公司提供的持有账户的会员。数据集包括客户账户、人口统计、客户参与度和财务数据。每个数据集包括大约250,000个示例，具有88个特征(71个数字特征和17个名义特征)。数据清洗使用第四节中详细介绍的标准来提高预测精度。图4显示了12个数据集上几乎相同的客户和帐户任期分布数据。可以推断，约90%的客户的账户期限为5 - 12个月，平均期限≈7.6个月。因此，大多数客户在不到一年的时间内关闭了他们的账户。因此，我们采取了合理的方法，只保留开了6个月以上的客户账户，也就是说，我们取消了所有最近开的账户。

6.因果分析结果

  图6显示了基于影响流失率的假设的实证结果和相关因果图:

  • 高限额账户余额可能会影响客户的流失，因为与高账户余额的客户相比，限额较低(账户余额)的用户可能不会忠诚于养老基金。
  • 账户余额变动金额可能影响客户任期。毕竟，客户任期通常是基于账户余额的。账户余额本身可能会影响churn。在大多数养老基金中，账户余额不应低于1500美元，低账户余额通常代表着一个可能愿意退出的不活跃客户。
  • 账户余额与性别之间的级联关系表明，性别会影响账户余额，进而间接影响账户流失率。
  • 账户增长和高余额变化可能直接影响动荡。没有变化账户增长表明大多数账户停止了业务，从而增加了churn倾向。

  通过在改变目标处理的同时保持其他潜在影响不变，我们根据初始假设确定了对churn结果的因果处理影响。线性回归估计表明，效应= - 0.026298对应于客户拥有较高的账户期限时，客户流失概率降低≈2.5%。来检验我们的假设，以便在假设正确的情况下，新的估计效应不应发生显著变化。因此，我们应用随机共同原因(RCC)[20]，通过在数据集中加入独立随机变量res random作为共同原因来反驳得到的估计。反驳方法的结果= - 0.026299，与估计结果基本一致。因此，我们可以确认，高客户任期是客户流失结果的一个因果特征的假设是正确的。

七.总结

  作者设计了一个新的客户流失倾向模型，并将其与贝叶斯因果网络集成。在预处理中，对不平衡的流失和非流失类别进行了调整，然后将所提出的DFF NN方法与十种当前最佳实践的流失预测分类器进行了准确性比较。尽管XGBoost在AUC方面表现出色，但DFF NN在测试数据上获得了所有考虑模型中最高的准确性，并获得了可比较的AUC。
  研究人员分析了一个特定金融数据集中可能导致客户流失的原因，该数据集是在养老基金公司创建的。因果分析结果确认了最近的社保缴纳、年度报告偏好变化、账户增长和余额金额是与客户流失有高度相关性的混杂因素。根据专家知识，对于拥有活跃账户超过1年的客户，流失率可以降低约3%。
  这项工作的自然发展是扩展模式挖掘技术，使用较小的结果窗口进行研究，以获得更高效的预测结果。此外，还应该研究基于反事实因果分析的不同方法来确定流失原因。
